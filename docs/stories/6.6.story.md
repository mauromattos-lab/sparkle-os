# Story 6.6 — Enriquecimento do Briefing com Catálogo de Produtos

**Epic:** 6 — Canal de Blog AEO (Ghost CMS)
**Status:** Done
**Prioridade:** P2
**Executor:** @dev
**Quality Gate:** @qa
**Estimativa:** M
**Depende de:** 6.2, 5.6

---

## Descrição

Como Luiza, quero que os artigos gerados pelo Content Engine referenciem produtos reais da minha loja com links diretos, para que os leitores possam comprar os produtos mencionados no blog e o conteúdo tenha mais valor comercial.

Hoje os agentes geram artigos com links placeholder (`[texto](link)`) porque não têm acesso ao catálogo real. A solução é buscar produtos relevantes da NuvemShop API antes da geração e injetá-los no briefing do squad — permitindo que Lyra (agente de escrita) insira referências reais com URLs corretas.

---

## Acceptance Criteria

- [x] AC1: Antes de iniciar o pipeline, o sistema busca até 10 produtos da loja via `GET /v1/{store_id}/products` (NuvemShop API), ordenados por mais recentes
- [x] AC2: Os produtos são formatados como contexto e injetados no `client-context.md` ou passados diretamente como parte do prompt do squad
- [x] AC3: O agente Lyra (escrita) recebe os produtos no contexto e pode referenciar nome + URL canônica do produto no artigo
- [x] AC4: Links gerados pelo agente usam URLs reais (`canonical_url` da NuvemShop), não placeholders
- [x] AC5: Se a API da NuvemShop falhar ou não houver token configurado, o pipeline continua normalmente sem produtos (graceful degradation)
- [x] AC6: O `markdownToHtml` preserva links com URLs reais (`https://`) e descarta apenas os placeholders

---

## Escopo

**IN:**
- Busca de produtos da NuvemShop antes do pipeline
- Injeção de contexto de produtos no briefing do squad
- Preservação de links reais no publisher (já parcialmente feito)

**OUT:**
- Seleção inteligente de produtos por relevância de tópico (evolução futura)
- Links para categorias ou coleções (apenas produtos por agora)
- Integração com outros canais além de NuvemShop

## Dev Notes

### Fluxo de enriquecimento

```
1. pipeline-runner.ts: antes de chamar o squad, busca produtos
   GET /v1/{clientId}/products?per_page=10&sort_by=updated_at

2. Formata como bloco de contexto:
   ## Produtos da Loja (para referência no artigo)
   - Choker Illusion Corrente Manu Banhada à Prata: https://plakaacessorios.com/produtos/choker-illusion-...
   - Colar Gotas Eye Banhado à Prata: https://plakaacessorios.com/produtos/...
   ...

3. Injeta no prompt do squad como contexto adicional
   (append no userMessage ou no systemPrompt do agente Lyra)
```

### Arquivo de configuração do cliente

O `clients/plaka/config.yaml` já possui `NUVEMSHOP_USER_ID` e `NUVEMSHOP_ACCESS_TOKEN`. O enriquecedor usa esses valores.

### Tratamento de links no publisher

`nuvemshop-publisher.ts` → `markdownToHtml`:
- Links com `https://` → preservar como `<a href="...">`
- Links sem URL válida → manter só o texto (já implementado em 6.6 prep)
