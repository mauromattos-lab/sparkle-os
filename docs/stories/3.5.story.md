# Story 3.5 — Interface de Consulta do Cérebro para Agentes

## User Story
Como agente AIOX (Dex, Quinn, Aria, etc.) ou sistema SparkleOS, quero um cliente TypeScript tipado para consultar o Cérebro Coletivo, para que eu possa recuperar insights relevantes ao meu contexto atual sem precisar construir chamadas HTTP manualmente.

## Acceptance Criteria

- [ ] 1. `packages/brain-client/` criado como package pnpm no monorepo com `package.json`, `tsconfig.json` e entrada principal `src/index.ts`. Exporta `BrainClient` como named export.
- [ ] 2. `BrainClient` class implementada com os métodos: `search(query, options?)`, `getInsight(id)`, `listInsights(filters?)`, `ingest(input)`. Todos retornam `Promise` com tipos do Cérebro (`Insight`, `SearchResult[]`).
- [ ] 3. `BrainClient` recebe `baseUrl` e `apiKey?` no construtor — usa `fetch` nativo (Node.js 22). Erros HTTP mapeados para erros TypeScript descritivos (ex: `BrainClientError` com `status` e `message`).
- [ ] 4. Método `search(query, options?)` aceita os mesmos parâmetros da `POST /brain/insights/search` (Story 3.3) e retorna `{ results: SearchResult[] }`. Parâmetro `options` é totalmente opcional com defaults idênticos ao servidor (threshold=0.75, limit=10, statusFilter=['validated','applied']).
- [ ] 5. Método `getContext(taskDescription, options?)` implementado como abstração de alto nível: chama `search` com `taskDescription` como query, retorna apenas os campos `id`, `content`, `source`, `confidenceLevel`, `similarity` — otimizado para injeção em prompts de agentes.
- [ ] 6. Testes unitários em `packages/brain-client/src/client.test.ts` — mockar `fetch` — cobrindo: `search` com resposta válida, `search` com query inválida (erro 400), `getInsight` 404, `getContext` retorna campos reduzidos, erro de rede (fetch rejeita).
- [ ] 7. `packages/brain-client/` referenciado no `pnpm-workspace.yaml` (ou `package.json` raiz) e importável por outros packages via `@sparkle/brain-client`. Verificar que `pnpm install` resolve o package localmente.
- [ ] 8. SOP criado em `docs/sops/sop-agentes-consultar-cerebro.md` — como agentes integram o `BrainClient`: inicializar, chamar `getContext`, injetar resultado em prompt, exemplo completo TypeScript.

## Complexidade
**S (Pequeno)** — Novo package TypeScript puro que wraps chamadas HTTP. Sem banco de dados, sem embeddings, sem nova infra. A maior complexidade é o setup do package pnpm e garantir que os tipos estejam corretos sem criar dependência circular com `packages/brain/`.

## Riscos

- **Tipos duplicados vs. import circular:** `packages/brain/` tem os tipos em `src/types/insight.ts`. Importar diretamente de `packages/brain/` criaria dependência circular (servidor ← client). Solução: `brain-client` define seus próprios tipos compatíveis OU `packages/brain/` exporta tipos separados em um sub-package. Para v1, copiar os tipos necessários em `packages/brain-client/src/types.ts`.
- **pnpm workspace config:** Verificar se `packages/*` já está no `pnpm-workspace.yaml` antes de adicionar entry.
- **fetch em Node.js 22:** `fetch` é nativo desde Node.js 18 — sem necessidade de polyfill. Confirmar que o tsconfig do package inclui `lib: ["ES2022"]` ou superior.

### BrainClient — Interface Pública

```typescript
// packages/brain-client/src/client.ts

interface BrainClientOptions {
  baseUrl: string;      // ex: 'http://localhost:3003'
  apiKey?: string;      // Authorization header opcional
}

interface SearchOptions {
  limit?: number;               // default: 10
  threshold?: number;           // default: 0.75
  statusFilter?: InsightStatus[];  // default: ['validated', 'applied']
  minConfidence?: ConfidenceLevel;
}

interface ContextEntry {
  id: string;
  content: string;
  source: InsightSource;
  confidenceLevel: ConfidenceLevel;
  similarity: number;
}

class BrainClient {
  constructor(options: BrainClientOptions)
  
  search(query: string, options?: SearchOptions): Promise<{ results: SearchResult[] }>
  getInsight(id: string): Promise<Insight>
  listInsights(filters?: ListFilters): Promise<{ data: Insight[]; total: number; page: number }>
  ingest(input: InsightInput): Promise<Insight>
  
  // Abstração de alto nível para agentes
  getContext(taskDescription: string, options?: SearchOptions): Promise<ContextEntry[]>
}
```

### BrainClientError

```typescript
// packages/brain-client/src/errors.ts
export class BrainClientError extends Error {
  constructor(
    message: string,
    public readonly status: number,
    public readonly body?: unknown
  ) {
    super(message);
    this.name = 'BrainClientError';
  }
}
```

### Exemplo de Uso (para SOP)

```typescript
import { BrainClient } from '@sparkle/brain-client';

const brain = new BrainClient({
  baseUrl: process.env.BRAIN_URL ?? 'http://localhost:3003',
});

// Uso básico — busca semântica
const { results } = await brain.search('taxa de escalação Zenya');

// Uso de agente — getContext injeta resultado em prompt
const context = await brain.getContext(
  'Preciso melhorar o fluxo de atendimento inicial'
);
const contextStr = context
  .map(c => `[${c.source}] ${c.content}`)
  .join('\n');

const prompt = `Contexto do Cérebro:\n${contextStr}\n\nTarefa: ...`;
```
