# Story 5.4 — Integração Pinterest

**Epic:** 5 — Content Engine: Automação e Integração
**Status:** Done
**Prioridade:** P2
**Executor:** @dev
**Estimativa:** M
**Depende de:** 5.2

---

## Descrição

Como Mauro, quero que após minha aprovação no Cockpit o pin seja publicado automaticamente no Pinterest da Plaka, com a imagem do Drive, título, descrição e hashtags gerados pela Vista, sem intervenção manual.

---

## Acceptance Criteria

- [ ] AC1: Após aprovação no Cockpit, o pin é criado no Pinterest da Plaka via API
- [ ] AC2: Pin inclui: imagem do Drive (selecionada pela Vista), título, descrição, link para o post do blog
- [ ] AC3: Pin é publicado no board correto (configurável via variável de ambiente `PINTEREST_BOARD_ID`)
- [ ] AC4: Status no Cockpit atualiza para `pin_publicado` com link ao pin
- [ ] AC5: Em caso de falha na API Pinterest, status vai para `erro_pin` sem bloquear publicação do blog
- [ ] AC6: Credenciais Pinterest armazenadas como variáveis de ambiente (`PINTEREST_ACCESS_TOKEN`)

---

## Escopo

**IN:**
- Publicação via Pinterest API v5 (endpoint de criação de pins)
- Imagem: download do Google Drive via link compartilhado
- Link do pin aponta para o post publicado no blog NuvemShop

**OUT:**
- Múltiplos boards por tópico (board único por enquanto)
- Pinterest Analytics
- Pins agendados (publicação imediata após aprovação)

---

## Notas Técnicas

- Pinterest API v5: `POST /pins`
- Imagem: fazer download do Drive antes de enviar (Pinterest aceita URL ou upload direto)
- Falha no Pinterest não bloqueia publicação do blog — são independentes
- Referência: Pinterest Developer docs
