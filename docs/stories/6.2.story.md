# Story 6.2 — Ghost Publisher no Content Engine

**Epic:** 6 — Canal de Blog AEO (Ghost CMS)
**Status:** Done
**Prioridade:** P1
**Executor:** @dev
**Estimativa:** M
**Depende de:** 6.1 ✅ Done

---

## Descrição

Como Mauro, quero que após minha aprovação no Cockpit o post seja publicado automaticamente no Ghost CMS (em vez da NuvemShop), para que os artigos diários da Plaka fiquem disponíveis no blog com Schema AEO correto e sem nenhuma ação manual minha.

---

## Acceptance Criteria

- [x] AC1: Após aprovação no Cockpit, o post é publicado no Ghost via Admin API (`POST /ghost/api/admin/posts/`)
- [x] AC2: Post publicado inclui: título, corpo em HTML (convertido do Markdown do Lyra), slug gerado a partir do título
- [x] AC3: Post publicado inclui `codeinjection_head` com JSON-LD Schema `BlogPosting` válido (headline, author, publisher, datePublished, inLanguage)
- [x] AC4: Status no Cockpit atualiza para `publicado` com link direto ao post no Ghost
- [x] AC5: Em caso de falha na Ghost API, status vai para `erro_publicacao` com mensagem descritiva logada
- [x] AC6: Credenciais Ghost armazenadas como variáveis de ambiente (`GHOST_API_URL`, `GHOST_ADMIN_API_KEY`)
- [x] AC7: O `publication-orchestrator.ts` chama `publishToGhost` no lugar de `publishToNuvemShop` — NuvemShop publisher permanece no código mas não é chamado (remoção em Story 6.4)

---
