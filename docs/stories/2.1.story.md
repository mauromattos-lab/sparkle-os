# Story 2.1 — Inventário dos Fluxos n8n da Zenya

## User Story
Como agente AIOS, quero um inventário completo e estruturado de todos os fluxos n8n da Zenya — com nome, propósito, inputs, outputs, dependências e status operacional — para que qualquer agente possa entender o que a Zenya faz sem precisar acessar o n8n diretamente, e para que decisões de integração, melhoria e provisionamento sejam baseadas em dados reais.

## Acceptance Criteria

- [x] 1. Todos os fluxos n8n da Zenya catalogados em `docs/zenya/FLOW-INVENTORY.md` — mínimo esperado: ~15 fluxos
- [x] 2. Cada fluxo documentado com: ID no n8n, nome, propósito, trigger (webhook/schedule/manual), nodes principais, inputs esperados, outputs gerados, dependências externas (Chatwoot, Z-API, AI, outros) e status atual (ativo/pausado/em desenvolvimento)
- [x] 3. Mapa de relacionamentos entre fluxos identificado — quais fluxos chamam outros, quais compartilham recursos
- [x] 4. IP da Zenya identificado e sinalizado: lore, personalidade, arquivos de configuração sensíveis — listados em seção separada `## IP Protegido` dentro do inventário, sem expor o conteúdo bruto
- [x] 5. Fluxos classificados por função: `atendimento`, `qualificação`, `handoff`, `notificação`, `admin`, `outro`
- [x] 6. Gaps e riscos identificados: fluxos sem documentação, dependências frágeis, pontos únicos de falha — listados em `## Gaps e Riscos`
- [x] 7. Inventário consultável por qualquer agente AIOS via arquivo em `docs/zenya/` — sem necessidade de acesso ao n8n

## Escopo

**IN (esta story entrega):**
- `docs/zenya/FLOW-INVENTORY.md` — inventário completo dos fluxos
- `docs/zenya/ZENYA-CONTEXT.md` — contexto operacional: o que é a Zenya, como opera, quem são os clientes, como o sistema funciona como um todo
- Seção `## IP Protegido` no inventário — sinaliza o que é IP sem expor conteúdo
- Seção `## Mapa de Relacionamentos` — quais fluxos se comunicam
- Seção `## Gaps e Riscos` — observações do analista

**OUT (não inclui):**
- Modificação de nenhum fluxo n8n
- Versionamento ou backup dos fluxos (→ Story 2.2)
- Integração formal como Núcleo SparkleOS (→ Story 2.3)
- Qualquer código de aplicação
- Credenciais, tokens ou configurações sensíveis no documento

## Complexidade
**M (Médio)** — Requer acesso ao n8n para inspeção visual, análise de cada fluxo e síntese estruturada. Volume de ~15 fluxos com relacionamentos pode revelar complexidade não prevista.

## Riscos
- Acesso ao n8n na VPS pode requerer credenciais de Mauro — se @analyst não tiver acesso direto, escalar para Mauro antes de iniciar
- Fluxos podem estar mais interconectados do que o esperado — o mapa de relacionamentos pode ser complexo
- IP da Zenya (lore, personalidade) pode estar espalhado em variáveis de ambiente, nodes de código JS ou arquivos externos — inventário deve mapear onde está, não transcrever o conteúdo

### Sobre a Zenya
[Source: docs/architecture.md §2.1]
- Zenya é uma atendente IA no WhatsApp com ~15 fluxos n8n
- Stack: n8n + Chatwoot fazer.ai + Z-API
- Roda na VPS Hostinger KVM2 via Coolify (mesmo servidor que hospedará o SparkleOS Core)
- Chatwoot fazer.ai já abstrai o provider WhatsApp (Z-API / Baileys / API oficial)

### Arquitetura Futura da Zenya no SparkleOS
[Source: docs/architecture.md §6 — Zenya Adapter]
- A Zenya será encapsulada em `organs/zenya/` como Zenya Adapter
- O resto do sistema não conhecerá n8n diretamente — tudo via adapter
- API interna expõe: `GET /nucleus/zenya/flows` (inventário) e `POST /nucleus/zenya/flows/{id}/clone`
- **Esta story produz o insumo que define o conteúdo desse endpoint**

### Modelo de Dados Relevante
[Source: docs/architecture.md §4 — ZenyaClient]
```typescript
interface ZenyaClient {
  id: string;
  externalId: string;           // Chatwoot inbox ID
  name: string;
  whatsappNumber: string;
  n8nWorkflowIds: string[];     // ← Esta story deve popular com IDs reais
  chatwootInboxId: number;
  status: 'active' | 'paused' | 'offboarded';
  dataIsolationKey: string;
  provisionedAt: string;
  provisionedBy: string;
  metadata: Record<string, unknown>;
}
```
O campo `n8nWorkflowIds` no modelo `ZenyaClient` será populado com IDs identificados neste inventário.
