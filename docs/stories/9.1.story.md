**Executor:** @dev

## Descrição

Criar o workflow n8n `plaka-daily-content` no servidor VPS com trigger cron diário às 09:00 BRT. O workflow executa os steps 0 a 6 do pipeline: gate de scheduling, briefing (Sage), escrita (Lyra), validação (Rex) com loop de revisão, e escalação para Mauro quando necessário.

O workflow chama a Claude API diretamente (modelo claude-sonnet-4-6) em cada step de agente, com os system prompts extraídos das tasks do squad (`daily-briefing.md`, `write-post.md`, `validate-post.md`, `rewrite-post.md`).

---

## Acceptance Criteria

**AC1 — Trigger cron:**
- Workflow configurado com cron expression `0 9 * * *` (09:00 UTC-3 / America/Sao_Paulo)
- Timezone configurado como `America/Sao_Paulo` no n8n
- Execução pode ser acionada manualmente para testes

**AC2 — Gate de scheduling (Step 0):**
- Workflow lê `squads/aeo-squad-plaka/data/posts-history.md` do repositório
- Verifica se já existe entrada com a data de hoje
- Se sim: para execução com log "Já existe post hoje — pulando"
- Se não: prossegue para Step 1

**AC3 — Sage gera briefing (Step 1):**
- Chama `POST https://api.anthropic.com/v1/messages` com model `claude-sonnet-4-6`
- System prompt: conteúdo completo de `squads/aeo-squad-plaka/tasks/daily-briefing.md`
- Contexto injetado: data atual + conteúdo de `posts-history.md` (últimos 10 posts)
- Output parsed: tópico, bloco_conteudo, palavras_chave, angulo_narrativo

**AC4 — Lyra escreve post (Step 2):**
- Chama Claude API com system prompt de `tasks/write-post.md`
- Contexto: briefing do Step 1 + `data/plaka-context.md` + últimos 5 posts do histórico
- Output: post completo em Markdown (título, intro, corpo, FAQ, CTA)
- Mínimo 800 palavras

**AC5 — Rex valida (Step 3):**
- Chama Claude API com system prompt de `tasks/validate-post.md`
- Avalia post do Step 2
- Output parsed: `veredicto` (APROVADO | REVISAO | ESCALADO) + `feedback` (se REVISAO)

**AC6 — Loop de revisão (Steps 4-5):**
- Se `veredicto == REVISAO`: Lyra reescreve com feedback do Rex (Step 4)
- Rex revalida (Step 5)
- Máximo 2 iterações — se 2ª rejeição: veredito = ESCALADO

**AC7 — Escalação (Step 6):**
- Se `veredicto == ESCALADO`: workflow para e registra no log do n8n
- Notificação de escalação tratada na Story 9.3
- Workflow marca execução como "requires_human_review"

**AC8 — Tratamento de erros:**
- Falha na Claude API: retry automático 2x com backoff de 30s
- Falha após retries: execução marcada como erro, log no n8n, não tenta publicar

## Notas Técnicas

**Leitura de arquivos no n8n:**
O VPS tem o repositório clonado. Usar `Code node` ou `Execute Command node` para ler os arquivos:
```bash
cat /path/to/SparkleOS/squads/aeo-squad-plaka/tasks/daily-briefing.md
cat /path/to/SparkleOS/squads/aeo-squad-plaka/data/posts-history.md
cat /path/to/SparkleOS/squads/aeo-squad-plaka/data/plaka-context.md
```

**Claude API — formato da chamada:**
```json
{
  "model": "claude-sonnet-4-6",
  "max_tokens": 4096,
  "system": "<conteúdo do task file>",
  "messages": [{ "role": "user", "content": "<contexto injetado>" }]
}
```

**Variáveis n8n necessárias:**
- `ANTHROPIC_API_KEY` — chave da API Claude
- `SPARKLE_REPO_PATH` — path do repo no VPS (ex: `/opt/sparkle`)
