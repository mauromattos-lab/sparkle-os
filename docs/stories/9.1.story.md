# Story 9.1 — GitHub Actions: Workflow + Pipeline Sage→Rex

**Epic:** [Epic 9](./EPIC-9-INDEX.md)
**Status:** Draft
**Prioridade:** P1 — Blocker
**Executor:** @dev
**Estimativa:** S (2-3h)

---

## Descrição

Criar o GitHub Actions workflow `plaka-daily-content.yml` com cron diário às 09:00 BRT e o script `daily-pipeline.mjs` que executa os steps 0 a 6: gate de scheduling, briefing (Sage), escrita (Lyra), validação (Rex) com loop de revisão, e saída por escalação quando necessário.

O workflow roda em `ubuntu-latest`, faz checkout do repo (acesso nativo a posts-history.md e tasks), instala dependências Node.js e executa o script. Cada step de agente chama a Claude API diretamente com os system prompts extraídos das tasks do squad.

---

## Acceptance Criteria

**AC1 — GitHub Actions workflow:**
- Arquivo `.github/workflows/plaka-daily-content.yml` criado
- Trigger `schedule` com cron `0 12 * * *` (09:00 BRT = UTC−3)
- Trigger `workflow_dispatch` para execução manual com input opcional `--force`
- Job roda em `ubuntu-latest`
- Usa `actions/checkout@v4` e `actions/setup-node@v4` (Node 20)
- Instala dependências: `cd packages/content-engine/scripts && npm install`
- Secrets injetados como variáveis de ambiente no step de execução

**AC2 — Gate de scheduling (Step 0):**
- Script lê `squads/aeo-squad-plaka/data/posts-history.md` (disponível via checkout)
- Verifica se há linha com a data de hoje (formato `YYYY-MM-DD`)
- Se sim: `process.exit(0)` com log "Já existe post hoje — pulando"
- Se `--force` passado via `workflow_dispatch`: ignora o gate

**AC3 — Sage gera briefing (Step 1):**
- Chama `POST https://api.anthropic.com/v1/messages`
- Model: `claude-sonnet-4-6`, max_tokens: 2048
- System prompt: lê `squads/aeo-squad-plaka/tasks/daily-briefing.md`
- Contexto injetado no user message: data atual + últimos 10 posts do histórico + sazonalidade
- Output parseado: `{ topico, bloco_conteudo, palavras_chave, angulo_narrativo }`

**AC4 — Lyra escreve post (Step 2):**
- Chama Claude API com system prompt de `tasks/write-post.md`
- Contexto: briefing do Step 1 + conteúdo de `data/plaka-context.md` + últimos 5 slugs do histórico
- Output: post completo em Markdown
- Mínimo 800 palavras — se abaixo: retry automático com instrução "expanda para pelo menos 800 palavras"

**AC5 — Rex valida (Step 3):**
- Chama Claude API com system prompt de `tasks/validate-post.md`
- Input: post do Step 2
- Output parseado: `{ veredicto: "APROVADO"|"REVISAO"|"ESCALADO", feedback: string }`

**AC6 — Loop de revisão (Steps 4-5):**
- Se `veredicto == "REVISAO"`: Lyra reescreve com feedback (Step 4) → Rex revalida (Step 5)
- Máximo 2 iterações
- Se 2ª rejeição: `veredicto = "ESCALADO"`

**AC7 — Saída por escalação (Step 6):**
- Se `veredicto == "ESCALADO"`: script seta output `needs_escalation=true` e encerra com `process.exit(0)` (não falha o job — é condição esperada)
- O step de notificação (Story 9.3) consome esse output

**AC8 — Tratamento de erros da Claude API:**
- Retry automático 2x com backoff de 30s para erros 429, 500, 503
- Após retries esgotados: `process.exit(1)` — job falha → GitHub envia email automático

## Notas Técnicas

**Estrutura do workflow:**
```yaml
name: Plaka Daily Content
on:
  schedule:
    - cron: '0 12 * * *'
  workflow_dispatch:
    inputs:
      force:
        description: 'Ignorar gate de 1 post/dia'
        type: boolean
        default: false

jobs:
  daily-content:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - name: Install dependencies
        run: cd packages/content-engine/scripts && npm install
      - name: Run daily pipeline
        run: node packages/content-engine/scripts/daily-pipeline.mjs
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
          GHOST_API_URL: ${{ secrets.GHOST_API_URL }}
          GHOST_ADMIN_API_KEY: ${{ secrets.GHOST_ADMIN_API_KEY }}
          NUVEMSHOP_ACCESS_TOKEN: ${{ secrets.NUVEMSHOP_ACCESS_TOKEN }}
          NUVEMSHOP_USER_ID: ${{ secrets.NUVEMSHOP_USER_ID }}
          ZAPI_INSTANCE_ID: ${{ secrets.ZAPI_INSTANCE_ID }}
          ZAPI_TOKEN: ${{ secrets.ZAPI_TOKEN }}
          MAURO_PHONE: ${{ secrets.MAURO_PHONE }}
          FORCE: ${{ inputs.force }}
```

**Chamada Claude API — utilitário reutilizável:**
```javascript
async function callClaude(systemPrompt, userMessage, maxTokens = 4096) {
  for (let attempt = 0; attempt < 3; attempt++) {
    const res = await fetch('https://api.anthropic.com/v1/messages', {
      method: 'POST',
      headers: {
        'x-api-key': process.env.ANTHROPIC_API_KEY,
        'anthropic-version': '2023-06-01',
        'content-type': 'application/json',
      },
      body: JSON.stringify({
        model: 'claude-sonnet-4-6',
        max_tokens: maxTokens,
        system: systemPrompt,
        messages: [{ role: 'user', content: userMessage }],
      }),
    });
    if (res.ok) return (await res.json()).content[0].text;
    if ([429, 500, 503].includes(res.status) && attempt < 2) {
      await new Promise(r => setTimeout(r, 30000 * (attempt + 1)));
      continue;
    }
    throw new Error(`Claude API ${res.status}: ${await res.text()}`);
  }
}
```
