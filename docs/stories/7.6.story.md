### ElevenLabs — geração de áudio
```typescript
// integrations/elevenlabs.ts
export async function generateAudio(ssml: string, apiKey: string): Promise<Buffer> {
  const response = await fetch(
    'https://api.elevenlabs.io/v1/text-to-speech/VOICE_ID',
    {
      method: 'POST',
      headers: {
        'xi-api-key': apiKey,
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({
        text: ssml,
        model_id: 'eleven_flash_v2_5',
        voice_settings: { stability: 0.5, similarity_boost: 0.75 },
      }),
    }
  );
  return Buffer.from(await response.arrayBuffer());
}
```

### Envio de áudio no Chatwoot (multipart/form-data)
```typescript
export async function sendAudioMessage(params: ChatwootParams, audioBuffer: Buffer) {
  const formData = new FormData();
  formData.append('attachments[]', new Blob([audioBuffer], { type: 'audio/mpeg' }), 'response.mp3');
  formData.append('message_type', 'outgoing');
  formData.append('content', ''); // mensagem vazia — só o áudio
  
  await fetch(
    `${params.url}/api/v1/accounts/${params.accountId}/conversations/${params.conversationId}/messages`,
    {
      method: 'POST',
      headers: { 'api_access_token': params.token },
      body: formData,
    }
  );
}
```

### SSML formatting via LLM

```typescript
const { text: ssml } = await generateText({

  prompt: text,

```

### Mensagem de fallback (AC6)
A mensagem de fallback deve ser a própria resposta em texto, sem qualquer indicação de erro técnico. O usuário não precisa saber que o áudio falhou — só precisa receber a resposta.

### Voice ID
O Voice ID da ElevenLabs deve ser configurável por tenant (campo na `zenya_tenants` ou em `active_tools`). Default: voz padrão da Zenya. Verificar qual Voice ID está sendo usado atualmente no n8n.
