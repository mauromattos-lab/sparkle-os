## Dev Notes

### Vercel AI SDK 5 — agent loop
```typescript
import { generateText } from 'ai';
import { openai } from '@ai-sdk/openai';

export async function runZenyaAgent(params: {
  tenantId: string;
  config: TenantConfig;
  message: string;
  phone: string;
  conversationId: string;
}) {
  const history = await loadConversationHistory(params.tenantId, params.phone, 50);
  const tools = createTenantTools(params.tenantId, params.config);

  const result = await generateText({
    model: openai('gpt-4.1'),
    maxSteps: 15,
    system: buildSystemPrompt(params.config),
    messages: [
      ...history,
      { role: 'user', content: params.message }
    ],
    tools,
  });

  await saveConversationHistory(params.tenantId, params.phone, params.message, result.text);
  return result.text;
}
```

### System prompt base (prompt.ts)
O `prompt.ts` deve conter apenas o template base da Zenya (personalidade, instruções gerais). O SOP específico do cliente vem de `config.system_prompt` e é concatenado. Isso permite evoluir a personalidade base sem afetar o SOP por cliente.

```typescript
export const ZENYA_BASE_PROMPT = `
Você é Zenya, uma atendente virtual...
[personalidade e instruções gerais aqui]

---
Contexto do cliente:
{{client_sop}}
`;

export function buildSystemPrompt(config: TenantConfig): string {
  return ZENYA_BASE_PROMPT.replace('{{client_sop}}', config.system_prompt);
}
```

### Chatwoot — enviar mensagem
```typescript
// integrations/chatwoot.ts
export async function sendMessage(params: {
  chatwootUrl: string;
  accountId: string;
  conversationId: string;
  content: string;
  token: string;
}) {
  await fetch(
    `${params.chatwootUrl}/api/v1/accounts/${params.accountId}/conversations/${params.conversationId}/messages`,
    {
      method: 'POST',
      headers: { 'api_access_token': params.token, 'Content-Type': 'application/json' },
      body: JSON.stringify({ content: params.content, message_type: 'outgoing' }),
    }
  );
}
```

### Memória — tabela existente vs nova
O n8n usa `n8n_historico_mensagens`. Para compatibilidade durante a migração paralela, criar uma nova tabela `zenya_conversation_history` com schema limpo, sem dependência das tabelas do n8n. O n8n continua usando as suas tabelas.

### Conexão entre webhook (7.1) e agent loop (7.3)
O worker (`worker/webhook.ts`) deve chamar o agent loop após enfileirar:
```
webhook recebe mensagem
  → queue.enqueue(message)
  → (async) agent.process(message, tenantId, ...)
  → chatwoot.sendMessage(result)
  → lock.release()
```
