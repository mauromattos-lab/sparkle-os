# Story 4.6 — Rastreamento de Custos

## User Story

Como Mauro, quero ver quanto estou gastando por agente e operação versus o meu orçamento de ~$550/mês, para que eu saiba se o sistema está dentro do custo previsto e qual agente está consumindo mais.

## Status
- [x] Draft
- [x] Ready
- [ ] InProgress
- [ ] InReview
- [ ] Done

## Acceptance Criteria

- [ ] 1. Painel `/cockpit/costs` exibe custo total do mês corrente vs. orçamento ($550/mês) — com indicador visual (verde < 80%, amarelo 80–100%, vermelho > 100%)
- [ ] 2. Breakdown de custo por agente no mês corrente — lista com agente e valor em dólares
- [ ] 3. Se não houver dados de custo ainda (sistema novo), exibe "Nenhum custo registrado este mês" — sem estado de erro
- [ ] 4. Painel acessível via navegação lateral do shell
- [ ] 5. Dados consumidos de `getCostSummary()`, `getBudgetStatus()` e `getCostByAgent()` de `@sparkle-os/core`

## Escopo

**IN (esta story entrega):**
- `packages/cockpit/src/services/costs.service.ts` — wrapper das funções de custo do core
- `packages/cockpit/src/routes/costs.ts` — painel HTML
- Testes unitários para `costs.service.ts` (mock de `@sparkle-os/core`)

**OUT (não inclui):**
- Histórico de custos por períodos anteriores — foco no mês corrente apenas nesta story
- Otimização ou recomendações de custo — cockpit é leitura, não ação
- Gráficos de tendência — texto + cor é suficiente

## Complexidade
**M (Médio)** — Mesmo risco da Story 4.3: core usa `postgres.js` (porta 5432 direta). Verificar conectividade.

## Riscos

- **DB do core (porta 5432):** Mesma situação da Story 4.3. `@sparkle-os/core` usa `postgres.js`. Se porta 5432 bloqueada, as funções falham. Tratar com graceful degradation: exibir "Dados de custo indisponíveis" em vez de quebrar.
- **Sistema sem dados:** Plataforma nova pode não ter registros de custo ainda. `getCostSummary()` pode retornar totais zerados — tratar como estado válido.

[Source: packages/core/src/costs/cost-router.ts + cost-store.ts]

```typescript
import { getCostSummary, getCostByAgent, getBudgetStatus } from '@sparkle-os/core';

// CostSummary para o mês corrente:
const summary = await getCostSummary('monthly', new Date().toISOString().slice(0, 10));
// → { period, totalCost: number, operationBreakdown: Record<string, number> }

// Por agente:
const byAgent = await getCostByAgent('monthly', new Date().toISOString().slice(0, 10));
// → AgentCostSummary[] → [{ agentId, totalCost, operationBreakdown }]

// Status do orçamento:
const budget = await getBudgetStatus();
// → BudgetStatus { budgetLimit: number, spent: number, remaining: number, percentUsed: number }
// budgetLimit configurado como $550 (NFR4 — docs/architecture.md)
```
