# Story 1.3 — Persistência de Contexto dos Agentes

## User Story
Como agente AIOS, quero persistir contexto de trabalho entre sessões, para que nenhuma decisão, raciocínio ou progresso se perca quando uma sessão termina.

## Acceptance Criteria
- [x] 1. Mecanismo de persistência implementado: Redis (72h TTL) para contexto ativo + Postgres para histórico permanente
- [x] 2. Agente consegue retomar trabalho de sessão anterior sem reconstrução manual de contexto — via API do Context Store
- [x] 3. Campos preservados: decisões em andamento, arquivos modificados, próximos passos, blockers, storyId ativo

## Technical Context

### Data Model — AgentContext (de `docs/architecture.md` §4)

```typescript
interface AgentContext {
  id: string;
  agentId: string;          // 'dev', 'architect', 'pm', etc.
  sessionId: string;
  storyId: string | null;
  workState: {
    currentTask: string;
    filesModified: string[];
    nextAction: string;
    blockers: string[];
  };
  decisionLog: Array<{
    decision: string;
    rationale: string;
    alternatives?: string[];
    timestamp: string;
  }>;
  createdAt: string;
  updatedAt: string;
}
```

### Decisão de Arquitetura (de @architect)
- **Redis (Coolify na VPS):** Cache quente com TTL 72h — recuperação < 5ms
- **Postgres (Supabase):** Armazenamento permanente para histórico e consulta por agentes diferentes
- **Estratégia:** Write-through — escreve Redis e Postgres simultaneamente

### API Endpoints Esperados (Hono — `packages/core`)
```
GET  /api/context/:agentId          # Retorna contexto ativo do agente
POST /api/context/:agentId          # Salva/atualiza contexto
GET  /api/context/:agentId/history  # Histórico completo (Postgres)
DELETE /api/context/:agentId        # Limpa contexto ativo
```

### Schema Postgres
```sql
CREATE TABLE agent_contexts (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  agent_id TEXT NOT NULL,
  session_id TEXT NOT NULL,
  story_id TEXT,
  work_state JSONB NOT NULL DEFAULT '{}',
  decision_log JSONB NOT NULL DEFAULT '[]',
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE INDEX ON agent_contexts(agent_id, created_at DESC);
```

### Redis Key Pattern
```
context:{agentId}:active  → JSON do AgentContext (TTL: 72h)
```

## Notas para @dev
- Context Store é o coração do sistema AI-native — implementar com cuidado
- Redis: usar client `ioredis` (melhor TypeScript support que `redis`)
- Postgres: queries via Drizzle ORM (sem Prisma — sem geração de código desnecessária para v1)
- O endpoint da API deve ser consumível por qualquer agente AIOS via fetch simples
- Incluir `updatedAt` com trigger Postgres ou update manual no service
- **Segurança:** API protegida por token interno (não OAuth) — ver Story 1.5 para config de credenciais
