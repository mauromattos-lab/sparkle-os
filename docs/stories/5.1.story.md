# Story 5.1 — Scheduler Diário do Squad

**Epic:** 5 — Content Engine: Automação e Integração
**Status:** InReview
**Prioridade:** P1 — Blocker
**Executor:** @dev
**Estimativa:** M

---

## Descrição

Como Mauro, quero que a pipeline do AEO Squad Plaka seja disparada automaticamente uma vez por dia, sem que eu precise iniciar nada manualmente, para que o conteúdo seja gerado de forma consistente.

---

## Acceptance Criteria

- [x] AC1: A pipeline `daily-content` do squad `aeo-squad-plaka` é executada automaticamente 1x por dia em horário configurável (default: 8h)
- [x] AC2: A execução roda Sage → Lyra → Rex → Vista em sequência
- [x] AC3: Se Rex rejeitar 2x, o item aparece no Cockpit com status "Escalado — requer atenção"
- [x] AC4: Se a pipeline falhar por erro técnico (API fora, timeout), o erro é logado e aparece no Cockpit
- [x] AC5: O scheduler não roda se já houver um post pendente de aprovação do dia anterior
- [x] AC6: Horário de execução é configurável via variável de ambiente `CONTENT_SCHEDULE_TIME`

---

## Escopo

**IN:**
- Cron job interno ao SparkleOS
- Execução da pipeline `squads/aeo-squad-plaka/workflows/daily-content.yaml`
- Log de execução com status de cada step
- Integração com o sistema de status do Cockpit

**OUT:**
- UI de configuração do horário (variável de ambiente é suficiente)
- Múltiplos squads por cliente (apenas Plaka neste momento)
- Retry automático em caso de falha de API

## Dev Notes (Aria — @architect)

**Decisão arquitetural:** Não existe scheduler no SparkleOS. Story é CREATE.

### Novo package: `packages/content-engine`

```
packages/content-engine/
  src/
    scheduler.ts        # node-cron — dispara pipeline no horário configurado
    pipeline-runner.ts  # executa squads/aeo-squad-plaka/workflows/daily-content.yaml
    status-store.ts     # escreve/lê status na tabela content_posts (Drizzle + Postgres)
  package.json          # dependência: node-cron, @sparkle-os/core (workspace:*)
```

**Dependência:** `node-cron` (única nova dependência)

**Tabela DB necessária** (criar via Drizzle migration em `@sparkle-os/core`):
```typescript
content_posts: {
  id, client_id, status, topic,
  title, meta, body_preview,
  image_desc, pin_copy,
  created_at, approved_at, published_at,
  blog_url, pin_url, error_msg
}
```

**Status possíveis** (AC3/AC4): `gerando` | `aguardando_aprovacao` | `aprovado` | `publicado` | `escalado` | `erro`

**Variável de ambiente:** `CONTENT_SCHEDULE_TIME` (formato cron: `"0 8 * * *"` = 8h diário)

**Guard de duplicata** (AC5): antes de disparar, verificar se existe `content_posts` com status `aguardando_aprovacao` criado no dia corrente.
