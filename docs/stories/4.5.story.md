# Story 4.5 — Painel do Cérebro Coletivo

## User Story

Como Mauro, quero ver o estado do ciclo de aprendizado do sistema — insights capturados, validados e aplicados — para que eu entenda se o Cérebro está funcionando e se o conhecimento está sendo aproveitado.

## Status
- [x] Draft
- [x] Ready
- [x] InProgress
- [x] InReview
- [x] Done

## Acceptance Criteria

- [x] 1. Painel `/cockpit/brain` exibe: total de insights, breakdown por status (raw / validado / aplicado / rejeitado) com indicadores visuais
- [x] 2. Taxa de completude do ciclo exibida (% de insights que chegaram a "aplicado") — campo `cycle.completionRate` do `DashboardData`
- [x] 3. Top 3 insights aplicados recentemente listados com: resumo, fonte e data de aplicação
- [x] 4. Indicador de saúde do Brain: verde se `GET /brain/health` retorna `status: "ok"`, vermelho caso contrário
- [x] 5. Dados consumidos de `GET /brain/dashboard` (porta 3003) — API já disponível desde Story 3.9

## Escopo

**IN (esta story entrega):**
- `packages/cockpit/src/services/brain.service.ts` — fetch de `/brain/health` e `/brain/dashboard`
- `packages/cockpit/src/routes/brain.ts` — painel HTML
- Testes unitários para `brain.service.ts` (mock fetch)

**OUT (não inclui):**
- Busca semântica de insights — escopo do Brain, não do cockpit
- Disparo do lifecycle pipeline — operação do Brain, não visibilidade
- Listagem completa de todos os insights — apenas resumo e top 3

## Complexidade
**B (Baixa)** — API do Brain já implementada e documentada (Story 3.9). Integração HTTP simples.

## Riscos

- **Brain não rodando:** Se `packages/brain` não estiver iniciado, as chamadas falham. `brain.service.ts` deve tratar isso com try/catch e retornar estado de erro claro (não travar cockpit).
- **Cache do Brain:** `GET /brain/dashboard` tem cache de 60s — dados podem ter até 1 minuto de atraso. Aceitável para visualização.

```
→ DashboardData {
    generatedAt: string (ISO)
    cacheHit: boolean

      total: number

      by_source: { zenya_operation, agent_research, mauro_input }

      validated: number

      rejected: number

    }
    insights_by_confidence: { authoritative, high, medium }
    top_applied: TopAppliedInsight[]  ← usar os 3 primeiros
    quality_distribution: QualityBucket[]
    duplicates: { total_duplicates, top_canonical[] }
  }
```
