# Story 1.8 — Rastreamento de Custo por Operação

## User Story
Como Mauro, quero visibilidade do custo de cada tipo de operação dos agentes, para que decisões de uso de ferramentas tenham fundamento financeiro real.

## Acceptance Criteria
- [ ] 1. Custo por tipo de operação rastreado: geração de texto (Claude API — tokens input/output), embeddings (Voyage AI), pesquisa web (EXA — por query), operações de banco (Supabase — dentro do free tier)
- [ ] 2. Custo por agente discriminado — cada operação tagged com `agentId`
- [ ] 3. Agregação por período disponível: diário, semanal, mensal

- [ ] 5. SOP de como revisar e otimizar custos quando necessário

## Technical Context

### Data Model — CostEvent
```typescript
interface CostEvent {
  id: string;
  agentId: string;
  operationType: 'llm_input' | 'llm_output' | 'embedding' | 'web_search' | 'storage_read' | 'storage_write';
  model?: string;             // 'claude-sonnet-4-6', 'voyage-3', etc.
  units: number;              // tokens, queries, bytes
  unitCost: number;           // custo por unidade em USD
  totalCost: number;          // units * unitCost em USD
  storyId: string | null;
  sessionId: string | null;
  metadata: Record<string, unknown>;
  createdAt: string;
}
```

### Preços de referência (2026)
| Operação | Modelo | Custo |
|----------|--------|-------|
| Input tokens | claude-sonnet-4-6 | $3/1M tokens |
| Output tokens | claude-sonnet-4-6 | $15/1M tokens |
| Embeddings | voyage-3 | $0.06/1M tokens |
| Web search | EXA | $0.005/query |
| Supabase | free tier | $0 (até 500MB, 2GB bandwidth) |

### API Endpoints (Hono — `packages/core`)
```
POST /api/costs/events          # Registra evento de custo
GET  /api/costs/summary?period=daily&date=2026-04-11  # Resumo agregado
GET  /api/costs/by-agent?period=monthly&month=2026-04  # Por agente
GET  /api/costs/budget          # Orçamento atual vs. gasto
```

### Schema Postgres
```sql
CREATE TABLE cost_events (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  agent_id TEXT NOT NULL,
  operation_type TEXT NOT NULL,
  model TEXT,
  units NUMERIC NOT NULL,
  unit_cost NUMERIC NOT NULL,
  total_cost NUMERIC NOT NULL,
  story_id TEXT,
  session_id TEXT,
  metadata JSONB DEFAULT '{}',
  created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE INDEX ON cost_events(agent_id, created_at DESC);
CREATE INDEX ON cost_events(created_at DESC);
```

### Orçamento atual
- Claude API: ~$550/mês (já pago)
- Meta: operações de pesquisa < $20/mês adicionais
- Threshold de alerta: se custo projetado no mês ultrapassar 90% do orçamento → escalação para Mauro (Story 1.6)

## Notas para @dev
- CostTracking deve ser leve — operações de registro não devem impactar performance do fluxo principal
- Usar `async fire-and-forget` para registrar custo — nunca bloquear fluxo principal por falha de custo
- Preços devem ser configuráveis via variável de ambiente ou config file (não hardcoded)
- Budget mensal configurável: `MONTHLY_BUDGET_USD=570` (referência: $550 Claude + $20 extras)
- Anthropic SDK emite token usage em cada chamada — capturar no response handler
