# Story 5.6 — Onboarding de Cliente

**Epic:** 5 — Content Engine: Automação e Integração
**Status:** Ready
**Prioridade:** P3
**Executor:** @dev
**Estimativa:** M
**Depende de:** 5.3, 5.4

---

## Descrição

Como Mauro, quero conseguir configurar o Content Engine para um novo cliente de e-commerce em menos de 30 minutos, para que quando a Julia indicar um novo cliente eu não precise reconstruir nada do zero.

---

## Acceptance Criteria

- [ ] AC1: Existe um arquivo de configuração por cliente (`clients/{client-id}/config.yaml`) com: nome, NuvemShop credentials, Pinterest credentials, Drive folder ID, horário do scheduler, voz da marca
- [ ] AC2: O squad usa o `client-context.md` do cliente ao invés do `plaka-context.md` hardcoded
- [ ] AC3: Um novo cliente pode ser adicionado sem alterar código — apenas criando o arquivo de configuração
- [ ] AC4: O Cockpit distingue posts de diferentes clientes quando há mais de um ativo
- [ ] AC5: Existe um template de `client-context.md` que Mauro preenche para cada novo cliente

---

## Escopo

**IN:**
- Estrutura de configuração por cliente
- Parametrização do squad para receber cliente como variável
- Template de onboarding para Mauro preencher
- Cockpit com seletor de cliente (quando houver mais de um)

**OUT:**
- UI de onboarding (arquivo de configuração é suficiente por agora)
- Billing por cliente
- Relatórios por cliente

---

## Valor de Negócio

Esta story transforma o Content Engine de **projeto pontual** em **produto escalável**. Cada novo cliente que a Julia indicar é configurado em 30 minutos, não em dias.

**ROI estimado:** se cada cliente paga R$500-1000/mês pelo Content Engine, a story se paga no primeiro cliente novo.
