## Story

**As** operador de onboarding de novos tenants,
**I want** um único script `seed-tenant.mjs --slug=X` substitua os 7 seeds duplicados (`seed-prime-tenant.mjs`, `seed-fun-personalize-tenant.mjs`, etc.),
**so that** onboarding de novo tenant exija ≤30min de trabalho operacional sem cópia-cola de scripts.

---

## Acceptance Criteria

1. Novo `packages/zenya/scripts/seed-tenant.mjs --slug=<slug>` parametrizado lê descriptor `docs/zenya/tenants/{slug}/seed.yaml`
2. Cada tenant ativo tem seu `seed.yaml` criado (7 tenants: prime, fun-personalize, hl, plaka, doceria, scar, ensinaja)
3. `seed.yaml` tem campos: `name`, `chatwoot_account_id`, `active_tools`, `admin_phones_env_var`, `admin_contacts_env_var`, `allowed_phones_env_var`, `prompt_path`
4. 7 seeds antigos (`seed-{tenant}-tenant.mjs`) deletados
5. `seed-common.mjs` mantém `applyTenantSeed` (canônico)
6. Cada tenant ativo re-seedado via novo script (validação md5 igual ao banco antes de aplicar)
7. Documentação Cap. 3 §1 + §3 atualizada com novo workflow

**Pattern proposto:** descriptor declarativo (yaml) + script único parametrizado.

**Não confundir:** o **prompt** continua em `docs/zenya/tenants/{slug}/prompt.md` (ADR-001 mantido). O `seed.yaml` adiciona ao lado, descrevendo a config NÃO-prompt do tenant.
