## Story

**As** dono Thainá (micropigmentação) e tenants futuros com Calendar ativo,
**I want** o bot envie lembretes automaticamente derivados dos agendamentos do Calendar (T-24h, T-2h, +manutenção pós-procedimento),
**so that** clientes não esqueçam sessões e cliente final receba lembretes no tom natural do bot (sem cadastro manual pelo dono).

## Acceptance Criteria

1. **Princípio P9 inviolável:** lembretes são DERIVADOS do Calendar, NUNCA cadastrados manualmente pelo dono. Worker periódico lê eventos futuros do tenant + janelas pré-definidas
2. Worker novo `packages/zenya/src/worker/outbound-reminders.ts`:
   - Lê Google Calendar de cada tenant com `active_tools.includes('google_calendar')`
   - Calcula janelas (T-24h, T-2h em horário comercial — silenciar 22h-08h)
   - Verifica filter: conversa não tem `agente-off`; cliente já mensageou ao menos 1x (existe contato no Chatwoot)
   - **Re-check Calendar antes de cada disparo** (Opção A — zero lembrete fantasma se cliente cancela)
   - Dispara via `sendMessage` (Chatwoot) — NÃO via Z-API direto
3. App PM2 dedicado `zenya-reminders` em `ecosystem.config.cjs`
4. Microcopy templates por tenant em `docs/zenya/tenants/{slug}/reminders.md`:
   - Template T-24h
   - Template T-2h
   - Template manutenção (caso Thainá)
5. Coluna `zenya_tenants.maintenance_windows JSONB` (mapping `procedimento → dias`) — config one-time onboarding caso Thainá
6. Bot processa **resposta do cliente** ao lembrete como mensagem normal (re-uso do path agente principal — sem flag especial)
7. Smoke específico cobre: cliente confirma / cancela / remarca após receber lembrete
8. Tabela de tracking `zenya_reminder_log` (audit) com `tenant_id, event_id, type (t24/t2/maintenance), sent_at, status`
