## Story

**As** Mauro pareando Z-API de um tenant novo,
**I want** o admin agent NÃO responder à rajada de mensagens sincronizadas pelo Z-API quando ele se conecta pela primeira vez,
**so that** eu não receba 5-10 mensagens de "Hoje: 0 conversas..." em sequência apenas porque o Z-API estava sincronizando histórico ao parear.

---

## Acceptance Criteria

1. Em `packages/zenya/src/agent/admin-agent.ts`, antes do agent rodar, validar:
   - Se `payload.created_at * 1000 < (process_boot_time + 60000)` E `payload.created_at < (NOW - 60s)` → SKIP (mensagem de sync histórico)
   - Caso contrário (mensagem do "presente"): processar normal
2. Constante `BOOT_GRACE_PERIOD_MS = 60000` (60s) — não configurável por enquanto (resposta direta Mauro 2026-04-25)
3. Quando filtro skip, log claro: `[admin] FILTERED burst — created_at=<ts>, age=<ms>ms` pra auditoria via `pm2 logs grep FILTERED`
4. **Mensagens do PRESENTE (`created_at` próximo a NOW)** JAMAIS são filtradas, mesmo nos primeiros 60s pós-boot
5. Smoke test cobre 3 cenários:
   - Boot recente (>60s) + msg histórica antiga → SKIP
   - Boot recente (<60s) + msg do presente → processa normal
   - Boot antigo + msg histórica → processa normal (filtro só atua nos primeiros 60s)
6. Sem regressão: admin agent continua respondendo mensagens reais imediatamente

## Dev Notes

**Cicatriz original:** Mauro pareou Z-API de Scar AI em 2026-04-24 e recebeu rajada de 5-10 mensagens de admin agent porque Z-API sincronizou histórico recente.

**Threshold 60s justificado** (UX Specialist Review §2.3):
- 5s muito agressivo (Mauro mandando exatamente no segundo do boot é filtrado por engano)
- 30s arriscado (Z-API às vezes sync >30s)
- **60s** é seguro (sync típico <30s; folga; raríssimo Mauro mandar dentro disso)
- 5min muito conservador (Mauro testando feature pós-deploy fica filtrado)

**Garantia adicional:** filtro **só atua sobre mensagens do passado**. Mensagem com `created_at = NOW` jamais filtrada.

**Referências canônicas:**
- Cap. 1 §4.7 (admin-agent)
- Frontend Spec §7 A10 (anti-pattern original)
- UX Specialist Review §2.3
