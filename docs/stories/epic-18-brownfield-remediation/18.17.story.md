## Acceptance Criteria

Roadmap em 3 fases (UX Specialist Review §2.7):

### R-Fase 1 — Leitura aprofundada (depende Story 18.16 fechada)
1. Tools admin novas pra consultas complexas:
   - `buscar_conversa_por_nome` (cruzar phone↔nome)
   - `historico_cliente_X` (todas conversas do cliente)

### R-Fase 2 — Consulta complexa (paralelo)
2. Mais tools:
   - `listar_clientes_recentes` (top N por última atividade)
   - `consultar_volume_diario` (gráfico textual de últimos 7d)

### R-Fase 3 — Ações (mínima — princípio P9 manda preferir automação)
3. 3 tools de ação (max):
   - `pausar_bot_conversa_X` (aplica `agente-off` numa conversa)
   - `reativar_bot_conversa_X` (remove `agente-off`)
   - `marcar_lead_quente` (label `lead-hot` no Chatwoot)
4. **Princípio P9 manda:** muitas "ações" deveriam ser automação. Avaliar caso a caso se cabe automação antes de implementar como ação manual
