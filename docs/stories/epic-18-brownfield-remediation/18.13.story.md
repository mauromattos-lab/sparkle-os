## Story

**As** operador preenchendo `allowed_phones`/`admin_phones` no seed de tenant novo,
**I want** validação automática de telefone brasileiro (com/sem 9, formatação variada) → E.164 normalizado,
**so that** PLAKA (Bahia/Rio com 9) não tenha duplicatas tipo `+5575992160632` e `+557592160632` ao mesmo tempo.

## Acceptance Criteria

1. `npm install libphonenumber-js` adicionada como dependency
2. `seed-common.mjs` exporta `normalizePhoneBR(raw): string` (E.164) e `normalizePhoneBRSoft(raw): string` (tolerante)
3. Script `packages/zenya/scripts/backfill-normalize-phones.mjs` aplica normalização nos `allowed_phones` + `admin_phones` existentes (dry-run primeiro)
4. Seeds passam usar `normalizePhoneBR()` antes de inserir no banco
5. Test cobre casos: `'11999999999'`, `'+5511999999999'`, `'(11) 99999-9999'`, `'5511999999999'` → todos retornam `+5511999999999`
