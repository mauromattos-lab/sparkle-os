## Story

**As** cliente conversando com a Zenya no WhatsApp,
**I want** quando eu segurar dedo em mensagem antiga + "responder" + escrever "quanto fica esse?", o bot entenda QUAL mensagem citei,
**so that** o bot não fique perdido com pronomes ("esse", "isso", "vai esse aí") e responda referenciando o contexto correto.

**Cicatriz original (2026-04-24):** Ryan (testador HL) reportou cenário onde reply citando produto antigo perde contexto. Grep confirmou zero processamento de `in_reply_to` no código atual. Atenuante atual = sorte (msg citada às vezes está nas últimas 50 lembradas).

## Acceptance Criteria

### Extração payload Chatwoot

1. **AC1:** `webhook.ts` extrai `content_attributes.in_reply_to` (ou equivalente — confirmar shape real). Esperado: `{ message_id: <number>, content: "<texto>", sender: { type: "contact"|"user" } }`
2. **AC2:** Se payload tem `in_reply_to` populado, busca msg citada no histórico Postgres (por `chatwoot_message_id` se disponível, senão por conteúdo+timestamp). Fallback: usa texto do payload se busca falhar.
3. **AC3:** Se payload **não** tem `in_reply_to` (caso comum) → fluxo inalterado. **Zero regressão.**

### Injeção no prompt LLM

4. **AC4:** Quando há msg citada, injeta no prompt como system note:
   > *"Cliente respondeu citando uma mensagem anterior: «[texto truncado em 300 chars]»"*
5. **AC5:** Se msg citada é da equipe humana (outgoing): prefixo *"Cliente respondeu citando uma mensagem da equipe: ..."*. Se é do próprio bot ou outro: marcar adequadamente.
6. **AC6:** Texto citado **NÃO vai pro `zenya_conversation_history`** — é contexto efêmero só pro turno atual. Evita duplicação em turnos futuros.

### Discovery do shape (pré-dev)

7. **AC7:** @dev valida empiricamente shape real do payload Chatwoot quando há quote. Opções:
   - (a) Mauro/Ryan testa cenário "responder mensagem" via WhatsApp → log do payload no core
   - (b) Mauro grava payload via Chatwoot API (`/conversations/X/messages`)
   - Documentar shape em `docs/zenya/engine/whatsapp-reply-payload.md`
8. **AC8:** Se Chatwoot **não** envia `in_reply_to` no webhook (só no API leitura), `webhook.ts` faz fetch extra à API quando detecta msg é reply. Trade-off: +1 API call por turno — aceitável se frequência baixa.

### Cobertura

9. **AC9:** Unit tests cobrindo:
   - Payload sem `in_reply_to` → fluxo normal, zero injeção
   - Payload com `in_reply_to` de msg de agente → prefixo "equipe" correto
   - Payload com `in_reply_to` de msg de cliente → prefixo "cliente" correto
   - Texto longo (>300 chars) → trunca com "..."
   - Msg citada ID inexistente → fallback texto do payload OU sem injeção
   - Zero regressão: `npm test` passa 100%
10. **AC10:** Smoke/REPL: cenário "cliente cita produto da lista anterior" → bot identifica corretamente qual produto. Testável via `chat-tenant.mjs --tenant=hl` simulando sequência.

### Documentação + cross-tenant

11. **AC11:** `docs/zenya/engine/quoted-message-context.md` publicado — feature, shape, comportamento, edge cases. CHANGELOG marca enhancement.
12. **AC12:** Gate @qa PASS ou PASS w/concerns
13. **AC13:** Após merge, feature liga **automaticamente** cross-tenant (não exige flag por tenant — enhancement transparente). Smoke cross-tenant confirma zero regressão.

## Dev Notes

**Bloqueador soft:** AC7 discovery precisa acontecer ANTES do dev real. Se Chatwoot não envia `in_reply_to` no webhook, código muda (precisa fetch API extra).

**Por que efêmero (AC6):** msg citada já está no histórico permanente. Re-injetar no histórico duplicaria ela quando turnos futuros consultarem `zenya_conversation_history`. System note transient resolve.

**Compatibilidade backward:** sem flag por tenant. Se shape do payload mudar no futuro Chatwoot, fallback é fluxo atual (zero quote = comportamento legado).

**Sinergia com 18.20:** independente. 18.20 é guard de saída (handoff); 18.21 é enrichment de entrada (quote). Podem ir em paralelo.

**Referências canônicas:**
- Frontend Spec §3 (densidade ≤2 — não inflaciona prompt)
- Cap. 1 §4 (agent-loop entry point)
- Cap. 2 §2 (zenya_conversation_history schema)

| Chatwoot muda shape em upgrade futuro | B | Degrade gracioso — if-absent → fluxo normal |
