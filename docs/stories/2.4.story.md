# Story 2.4 — Processo de Provisionamento de Novo Cliente Zenya

## User Story
Como agente AIOS, quero executar o provisionamento de um novo cliente Zenya com um único comando — clonando os flows n8n, criando o inbox no Chatwoot e registrando o cliente no Postgres com isolamento de dados — para que o processo seja repetível, rastreável e executável sem intervenção manual de Mauro (exceto aprovação inicial).

## Riscos
- Chatwoot API pode exigir configuração adicional para criar inbox com Z-API — verificar antes de implementar; se bloqueante, o step de Chatwoot pode ser substituído por um placeholder + instrução manual
- Clonagem de flow n8n via API pode não preservar credenciais/variáveis de ambiente — o clone pode precisar de configuração manual pós-clone; documentar no SOP
- Transação distribuída (n8n + Chatwoot + Postgres) não é atômica — implementar compensação ou saga simples

## Dev Notes

### Workflow de Provisionamento
[Source: docs/architecture.md §8 — Workflow 3]
```
Agente → POST /nucleus/zenya/clients
  → Core gera dataIsolationKey
  → Zenya Adapter inicia SOP:
      1. Duplica flow template no n8n
      2. Cria inbox no Chatwoot
      3. Registra em zenya_clients (Postgres)
  → Retorna cliente provisionado
```

### Schema `zenya_clients`
[Source: docs/architecture.md §9]
```sql
CREATE TABLE zenya_clients (
  id                   UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  external_id          TEXT UNIQUE,
  name                 TEXT NOT NULL,
  whatsapp_number      TEXT NOT NULL,
  n8n_workflow_ids     TEXT[] NOT NULL DEFAULT '{}',
  chatwoot_inbox_id    INTEGER,
  status               TEXT NOT NULL DEFAULT 'active',
  data_isolation_key   TEXT NOT NULL UNIQUE DEFAULT uuid_generate_v4()::TEXT,
  provisioned_at       TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  provisioned_by       TEXT NOT NULL,
  metadata             JSONB NOT NULL DEFAULT '{}'
);
```
Migration Drizzle a criar em `organs/zenya/src/db/migrations/`

### n8n Clone API
[Source: docs/architecture.md §7 — External APIs]
- Base URL interna: `http://localhost:5678/api/v1`
- Endpoint de clone (verificar na API do n8n v1): `POST /workflows/{id}/duplicate` ou `POST /workflows` com body do workflow original
- Flow template a clonar: `01. Secretária v3` (ID: `r3C1FMc6NIi6eCGI`)

### Chatwoot API
[Source: docs/architecture.md §7]
- Base URL: `https://[instancia].fazer.ai/api/v1`
- Auth: User Token (env var `CHATWOOT_USER_TOKEN`)
- Criar inbox: `POST /api/v1/accounts/{accountId}/inboxes`

### Compensação em Caso de Falha
Estratégia saga simplificada:
1. Clone n8n → se falhar: retornar erro, nada a desfazer
2. Criar Chatwoot inbox → se falhar: deletar clone n8n, retornar erro
3. INSERT Postgres → se falhar: deletar clone n8n + inbox Chatwoot, retornar erro
