## Dev Notes

**Schema a criar:**
```sql
-- Vincula usuários auth.users a clientes Zenya
CREATE TABLE zenya_client_users (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id     UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  client_id   UUID NOT NULL REFERENCES zenya_clients(id) ON DELETE CASCADE,
  created_at  TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE(user_id, client_id)
);

ALTER TABLE zenya_client_users ENABLE ROW LEVEL SECURITY;

CREATE POLICY client_user_self_only ON zenya_client_users
  FOR ALL TO authenticated
  USING (user_id = auth.uid());
```

**Helper de sessão:**
```typescript
// organs/zenya/src/auth/session.ts
export async function getClientSession(userId: string) {
  const row = await db.query(
    `SELECT zc.id, zc.data_isolation_key
     FROM zenya_client_users zcu
     JOIN zenya_clients zc ON zc.id = zcu.client_id
     WHERE zcu.user_id = $1 AND zc.status = 'active'
     LIMIT 1`,
    [userId]
  );
  if (!row.rows[0]) throw new Error('CLIENT_NOT_FOUND');
  return { clientId: row.rows[0].id, isolationKey: row.rows[0].data_isolation_key };
}
```

**Padrão de isolamento existente:** [Source: docs/zenya/ISOLATION-SPEC.md]
- `isolation_key` = `data_isolation_key` do `zenya_clients`
- Sempre usar `set_config('app.current_client_key', isolationKey, TRUE)` antes de queries
