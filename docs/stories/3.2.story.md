# Story 3.2 — Captura de Insights da Zenya

## User Story
Como Cérebro Coletivo, quero capturar automaticamente insights gerados pela operação da Zenya, para que o conhecimento acumulado no atendimento alimente o sistema sem intervenção manual.

## Complexidade
**A (Alto)** — Criação de novo package no monorepo do zero com integração de serviço externo (Voyage API), schema pgvector com índice HNSW, e lógica de canonicalização por similaridade vetorial. Múltiplas integrações novas: Supabase migrations, voyageai npm, Hono framework.

## Riscos

- **VOYAGE_API_KEY ausente no `.env`:** A ingestão falhará silenciosamente se a chave não estiver configurada — health check deve detectar isso
- **HNSW em base vazia:** Índice HNSW com `lists=0` pode causar erro em pgvector — canonicalização deve ter fallback para base com 0 registros (sem busca de similares, apenas salva)
- **Errata de dims:** `docs/architecture.md §4` ainda referencia 1536 dims — AC 7 é obrigatório para não conflitar com o schema real (1024)
- **pnpm workspace:** Novo package precisa ser registrado em `pnpm-workspace.yaml` — verificar padrão existente dos outros packages

### Canonicalização (AC 3)

```typescript
// Algoritmo — 4 passos (cerebro-coletivo.md §7)
async function checkCanonical(embedding: number[], db: DB): Promise<CanonicalCheck> {
  // 1. Se base vazia, skip (HNSW não funciona com 0 registros)
  const count = await db.countInsights({ status: { not: 'rejected' } });
  if (count === 0) return { isDuplicate: false };

  // 2. Buscar top 5 mais similares (excluir rejected)
  const similars = await db.searchSimilar(embedding, { limit: 5, exclude: ['rejected'] });
  
  // 3. Verificar threshold 0.92
  if (similars[0]?.similarity > 0.92) {
    return {
      isDuplicate: true,
      canonicalId: similars[0].id,
      similarityScore: similars[0].similarity,
    };
  }
  return { isDuplicate: false };
}
```

### ZenyaAdapter — AC 6

O adapter transforma dados do `zenya_execution_log` (tabela auxiliar criada nesta story) em `InsightInput`:

```typescript
// packages/brain/src/adapters/zenya.adapter.ts
export class ZenyaAdapter {
  fromExecutionLog(log: ZenyaExecutionLog): InsightInput {
    return {
      source: 'zenya_operation',
      nucleusId: 'zenya',
      sourceRef: log.executionId ?? log.flowId,
      content: `Fluxo "${log.flowName}" executado com status ${log.status}` +
               (log.durationMs ? ` em ${log.durationMs}ms` : '') +
               (log.errorMessage ? `. Erro: ${log.errorMessage}` : ''),
      tags: [log.flowName, log.status, 'zenya_execution'],
    };
  }
}
```
