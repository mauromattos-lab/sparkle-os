# Story 10.3 — Frontend: app Next.js do cockpit do cliente (telas base)

## User Story
Como cliente Zenya, quero um painel web onde faço login e visualizo as conversas e métricas do meu negócio, para que eu tenha visibilidade do que a Zenya está fazendo sem precisar perguntar ao Mauro.

- [x] Ready (GO — @po Pax, 2026-04-20)
- [ ] InProgress
- [ ] InReview
- [ ] Done

## Acceptance Criteria
- [ ] 1. App `apps/client-cockpit/` criada em Next.js 15 com App Router
- [ ] 2. Tela de login: campo e-mail + botão "Enviar magic link" com feedback visual
- [ ] 3. Dashboard: cards com "Conversas hoje", "Total de conversas", "Status do sistema"
- [ ] 4. Lista de conversas: tabela com data, preview da última mensagem, paginação
- [ ] 5. Sessão persistida: ao reabrir o browser, usuário permanece logado
- [ ] 6. Loading states e mensagens de erro visíveis em todas as telas
- [ ] 7. Responsivo: funciona em mobile (375px) e desktop (1280px)
- [ ] 8. Regra Sparkle-first: app testada com login real da instância SparkleOS
- [ ] 9. [Movido de 10.1 AC 6] Login com magic link funciona end-to-end: e-mail recebido → link clicado → dashboard carregado com dados reais do tenant SparkleOS

## Escopo

**IN (esta story entrega):**
- Estrutura do app `apps/client-cockpit/` com Next.js 15
- Tela de login com Supabase Auth (magic link)
- Dashboard com métricas (consome `GET /api/client/metrics`)
- Lista de conversas com paginação (consome `GET /api/client/conversations`)
- shadcn/ui como biblioteca de componentes (padrão do projeto)

**OUT (não inclui):**
- Detalhe de uma conversa (→ escopo futuro)
- Customização visual por cliente (logo, cores) (→ escopo futuro)
- Deploy (→ Story 10.4)

## Complexidade
**M (Médio)** — Next.js 15 e shadcn/ui já são padrão do projeto. Supabase Auth client-side é bem documentado.

## Dev Notes

**Estrutura do app:**
```
apps/client-cockpit/
├── src/
│   ├── app/
│   │   ├── layout.tsx              # Root layout + Supabase provider
│   │   ├── login/page.tsx          # Tela de login (magic link)
│   │   ├── dashboard/page.tsx      # Dashboard com métricas
│   │   └── conversations/page.tsx  # Lista de conversas
│   ├── components/
│   │   ├── ui/                     # shadcn/ui base
│   │   ├── MetricCard.tsx          # Card de métrica reutilizável
│   │   └── ConversationTable.tsx   # Tabela de conversas
│   ├── hooks/
│   │   ├── useMetrics.ts           # TanStack Query → /api/client/metrics
│   │   └── useConversations.ts     # TanStack Query → /api/client/conversations
│   └── lib/
│       ├── supabase.ts             # Supabase client (browser)
│       └── api.ts                  # HTTP client com Bearer token
├── package.json
└── next.config.ts
```
[Source: docs/architecture.md#10 Frontend Architecture]

**Auth client-side com Supabase:**
```typescript
// src/lib/supabase.ts
import { createBrowserClient } from '@supabase/ssr';

export const supabase = createBrowserClient(
  process.env.NEXT_PUBLIC_SUPABASE_URL!,
  process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!
);
```

**Hook de métricas:**
```typescript
// src/hooks/useMetrics.ts
export function useMetrics() {
  return useQuery({
    queryKey: ['client-metrics'],
    queryFn: () => api.get('/api/client/metrics'),
    refetchInterval: 60_000,
  });
}
```

**Stack frontend:** [Source: docs/architecture.md#3]
- Next.js 15 (App Router), shadcn/ui + Radix, TanStack Query v5, Tailwind CSS v4

**Supabase SSR package:** usar `@supabase/ssr` (não `@supabase/auth-helpers-nextjs` — deprecated)
