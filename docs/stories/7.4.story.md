## Descrição

**As a** usuário do WhatsApp,
**I want** receber respostas da Zenya em partes naturais com indicador de digitação,
**so that** a conversa pareça humana e não um dump de texto de uma só vez.

---

## Acceptance Criteria

- [x] AC1: `integrations/message-chunker.ts` chama `gpt-4.1-mini` com structured output (`z.object({ mensagens: z.array(z.string()) })`) para dividir texto em até 5 partes naturais
- [x] AC2: Para cada parte, o loop executa em sequência: ativa typing indicator → aguarda tempo calculado → envia mensagem → desativa typing → aguarda 1s
- [x] AC3: Tempo de digitação por parte calculado: `Math.min(60 * (chars / 4.5) / 150, 25)` segundos (máx 25s)
- [x] AC4: Loop é serial garantido — for...of com await, nunca Promise.all
- [x] AC5: Agent loop (7.3) usa `chunkAndSend` ao invés de `sendMessage` direto
- [x] AC6: 8 testes unitários — timing (4), serial order, typing calls, fallback, single chunk

## Dev Notes

### Implementação do chunker
```typescript
// integrations/message-chunker.ts
import { generateObject } from 'ai';
import { openai } from '@ai-sdk/openai';
import { z } from 'zod';

export async function chunkAndSend(params: {
  message: string;
  chatwootParams: ChatwootParams;
}) {
  const { object } = await generateObject({
    model: openai('gpt-4o-mini'),
    schema: z.object({
      mensagens: z.array(z.string()).max(5),
    }),
    system: `Divida a mensagem em partes naturais para WhatsApp.
      Máximo 5 partes. Respeite pontuação natural.
      Cada parte deve ser completa e fazer sentido sozinha.`,
    prompt: params.message,
  });

  for (const parte of object.mensagens) {
    const typingTime = Math.min(60 * (parte.length / 4.5) / 150, 25) * 1000;
    await setChatwootTyping(params.chatwootParams, true);
    await sleep(typingTime);
    await sendChatwootMessage(params.chatwootParams, parte);
    await sleep(1000);
  }
}

function sleep(ms: number) {
  return new Promise(resolve => setTimeout(resolve, ms));
}
```
