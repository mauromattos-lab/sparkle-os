# Story 4.3 — Fila de Decisões de Mauro

## User Story

Como Mauro, quero ver apenas as decisões que realmente precisam de mim — com contexto suficiente para decidir na hora — para que eu não precise procurar o que está esperando e não perca tempo com coisas que os agentes podem resolver sozinhos.

## Acceptance Criteria

- [ ] 1. Painel `/cockpit/decisions` lista todas as decisões pendentes com: título, contexto, quem pediu e quando
- [ ] 2. Badge com contagem de decisões pendentes visível na navegação lateral (ex: "Decisões (3)")
- [ ] 3. Cada decisão tem as opções disponíveis listadas quando presentes (campo `options` do modelo)
- [ ] 4. Painel vazio exibe mensagem "Nenhuma decisão pendente — tudo sob controle" (sem estado de erro)
- [ ] 5. Dados reais consumidos de `listPendingDecisions()` de `@sparkle-os/core` — sem mock

## Escopo

**IN (esta story entrega):**
- `packages/cockpit/src/services/decisions.service.ts` — wrapper de `listPendingDecisions()` do core
- `packages/cockpit/src/routes/decisions.ts` — painel HTML + badge
- Atualização do `shell.ts` para mostrar badge de contagem no link "Decisões"
- Testes unitários para `decisions.service.ts` (mock do módulo `@sparkle-os/core`)

**OUT (não inclui):**
- Criação ou resolução de decisões pelo cockpit — cockpit é read-only neste epic
- Notificações push ou alertas de nova decisão — polling 30s do shell é suficiente

## Complexidade
**M (Médio)** — Integração com package interno (`@sparkle-os/core`). Risco de conectividade com DB do core.

## Riscos

- **DB do core usa `postgres` (porta 5432):** O package `@sparkle-os/core` usa `postgres.js` (porta 5432 direta), diferente do Brain que migrou para Supabase REST API. Se a porta 5432 estiver bloqueada na rede do desenvolvedor, `listPendingDecisions()` vai falhar. @dev deve verificar conectividade e considerar migração do core para Supabase client (como foi feito no Brain) — registrar ADR se migrar.
- **Decisões inexistentes:** Sistema pode não ter nenhuma decisão registrada ainda. Painel vazio deve ser tratado graciosamente.
