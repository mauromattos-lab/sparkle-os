# Story 8.1 — Ghost: Tags, Taxonomia e Correções de Estrutura

## Descrição

Configurar a taxonomia de tags no Ghost CMS para os 6 blocos AEO do squad Plaka, aplicar as tags retroativamente nos 8 posts existentes via Admin API, atualizar a tagline do blog e criar a bio de autor. Essas correções destravam a arquitetura de clusters semânticos que os LLMs e crawlers usam para reconhecer autoridade temática.

**Por que é blocker:** sem tags, os posts são ilhas sem hierarquia. Sem hierarquia, não há cluster. Sem cluster, não há autoridade progressiva — que é o núcleo do AEO.

## Referências Técnicas

- Ghost Admin API: `/ghost/api/admin/tags/` (POST/GET)
- Ghost Admin API: `/ghost/api/admin/posts/{id}/` (PATCH com `tags`)
- Ghost Admin API: `/ghost/api/admin/settings/` (PUT)
- Ghost Admin API: `/ghost/api/admin/users/{id}/` (PUT)
- Credenciais: `packages/content-engine/.env` — `GHOST_API_URL` + `GHOST_ADMIN_API_KEY`
- Padrão JWT: `packages/content-engine/scripts/publish-dia-das-maes.mjs` (buildJwt function)

---

## Dev Notes

- Todos os scripts existentes em `packages/content-engine/scripts/` usam o mesmo padrão: carregar `.env`, buildJwt, fetch. Seguir o mesmo padrão.
- Ghost PATCH de post para adicionar tag requer enviar o array completo de tags (`tags: [{ slug: "cuidados" }]`) — não é append, é replace. Verificar tags existentes antes de PATCH.
- A tag `"como-limpar-semi-joia..."` tem slug com `-3` — provavelmente post duplicado ou versionado. Verificar via GET `/ghost/api/admin/posts/?filter=slug:como-limpar*` antes de assumir o slug.

---

## Risks

- **Risco:** Ghost API retorna erro se tag já existir com mesmo slug → Mitigação: GET antes de POST, skip se já existe
- **Risco:** PATCH de tags sobrescrever metadata existente nos posts → Mitigação: GET post completo, merge tags, PATCH com array completo
- **Rollback:** Tags podem ser removidas via DELETE `/ghost/api/admin/tags/{id}/` sem afetar conteúdo dos posts
