# Story 8.3 — Squad: Contextual Linking, Citações, Scheduling e Pillar Page

## Descrição

Fortalecer o squad AEO Plaka para a Semana 2 em diante com quatro melhorias de processo: (1) Lyra passa a incluir links contextuais no corpo dos posts; (2) Lyra inclui 1 citação externa verificável por post técnico; (3) o workflow bloqueia publicação de mais de 1 post por dia; (4) publicar a primeira pillar page agregando os 3 posts de cuidados existentes.

---

## Acceptance Criteria

**AC1 — `write-post.md` atualizado: contextual linking obrigatório:**
- [x] Task `squads/aeo-squad-plaka/tasks/write-post.md` atualizada com nova regra no checklist: `"[ ] Incluir 1-2 links internos contextuais no corpo do texto para posts relacionados existentes"`
- [x] Nova seção "Links Internos" adicionada às Regras de Linguagem da task com instrução: verificar `data/posts-history.md` e linkar posts relacionados dentro do parágrafo mais relevante (não só no rodapé)
- [x] Exemplo incluído: *"[banho de ouro](URL do post banhada-vs-folheada)"* dentro de parágrafo sobre espessura

**AC2 — `write-post.md` atualizado: citação externa obrigatória em posts técnicos:**
- [x] Checklist atualizado com: `"[ ] Incluir 1 citação externa verificável para posts com afirmações técnicas (dados de mercado, saúde, química)"`
- [x] Fontes recomendadas listadas na task: SBD (Sociedade Brasileira de Dermatologia), ABNT, INMETRO, publicações científicas sobre ligas metálicas
- [x] Regra: citação em rodapé de frase com link ou referência entre parênteses — não nota de rodapé formal

**AC3 — `daily-content.yaml` com gate de scheduling:**
- [x] Adicionado step `step-0-scheduling-check` no início do workflow antes do `step-1-briefing`
- [x] Lógica: verificar em `data/posts-history.md` se já existe entrada com `data_atual` — se sim, STOP com mensagem: *"Já existe um post publicado hoje ({data}). Agendar para amanhã."*
- [x] Documentado no workflow: o gate é soft (pode ser ignorado com flag `--force`) para situações especiais

**AC4 — Pillar page "Guia Completo de Cuidados com Semi Joias" publicada:**
- [x] Script `publish-pillar-cuidados.mjs` criado em `packages/content-engine/scripts/`
- [x] Título: *"Guia Completo de Cuidados com Semi Joias: Tudo o Que Você Precisa Saber"*
- [x] Estrutura: introdução geral + 1 seção H2 por sub-tópico de cuidados, cada seção com resumo + link para o post completo correspondente
- [x] Links internos para os 3 posts de cuidados existentes: limpar, pele verde/manchas, banho
- [x] Tag: `cuidados` configurada no script
- [x] Meta description orientada para buscas amplas: *"Tudo sobre cuidados com semi joias: como limpar, por que mancha, o que evita o desgaste do banho de ouro. Guia técnico completo — Plaka Acessórios."*
- [x] JSON-LD: BlogPosting com `articleSection: Cuidados` + FAQPage com as 3 perguntas do H2 final
- [x] Post publicado em `blog.plakaacessorios.com` — publicado em https://blog.plakaacessorios.com/guia-completo-de-cuidados-com-semi-joias/

**AC5 — `register-post.md` atualizado para registrar URL do post publicado:**
- [x] Adicionada coluna `URL` na tabela de registro do `posts-history.md` (URLs reais dos 7 posts existentes preenchidas)
- [x] Task `register-post.md` atualizada para preencher URL do post após publicação (necessário para contextual linking futuro)

---

## Scope

**IN:**
- Atualização das tasks `write-post.md` e `register-post.md` do squad
- Atualização do workflow `daily-content.yaml` (gate de scheduling)
- Publicação da pillar page de cuidados

**OUT:**
- Pillar pages das demais categorias (criar quando cada cluster tiver 3+ posts)
- Alterações no agente Lyra ou Rex (persona permanece — só as tasks mudam)
- Alterações no content-engine (pipeline automatizado)

## Risks

- **Risco:** Pillar page duplicar conteúdo dos posts específicos → Mitigação: pillar usa resumos de 80 palavras + link, não copia o conteúdo completo
- **Risco:** Scheduling gate bloquear execuções legítimas → Mitigação: flag `--force` disponível para override documentado
- **Rollback:** Tasks do squad são arquivos markdown — qualquer mudança é reversível via git
