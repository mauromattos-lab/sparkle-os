# Story 6.7 — AEO: Feature Image no Ghost Publisher

**Epic:** 6 — Canal de Blog AEO (Ghost CMS)
**Status:** InReview
**Prioridade:** P2
**Executor:** @dev
**Estimativa:** S
**Depende de:** 6.2 ✅ Done, 6.6 ✅ Done

---

## Descrição

Como Mauro, quero que os posts publicados no Ghost incluam uma imagem de destaque (`feature_image`) com alt text descritivo no padrão AEO, para que o Google AIO exiba thumbnails nos citation cards e os mecanismos de IA (Perplexity, ChatGPT Search) entendam visualmente o conteúdo — aumentando a taxa de seleção do blog em respostas de IA.

Hoje o `ghost-publisher.ts` publica sem `feature_image`, o que significa:
- Nenhum `og:image` gerado pelo Ghost (campo fica vazio)
- Schema `BlogPosting` sem `ImageObject` (penalização nos rich results)
- GPTBot e PerplexityBot sem sinal visual do conteúdo

O campo `imageDesc` (alt text) já é gerado pelo agente Vista e salvo no `ContentPost` — só não está sendo usado na publicação no Ghost. A imagem em si será obtida do catálogo de produtos da NuvemShop (fonte pública, CDN acessível), mesma API já integrada na Story 6.6.

---

## Acceptance Criteria

- [ ] AC1: `product-enricher.ts` exporta nova função `fetchFirstProductImageUrl(): Promise<string | null>` que retorna a URL da imagem do primeiro produto publicado com imagem disponível (`images[0].src`)
- [ ] AC2: `publication-orchestrator.ts` chama `fetchFirstProductImageUrl()` antes de `publishToGhost` e passa a URL como parâmetro opcional
- [ ] AC3: `ghost-publisher.ts` inclui `feature_image` e `feature_image_alt` no POST do Ghost quando disponíveis — publicação **não é bloqueada** se os campos estiverem ausentes (graceful degradation)
- [ ] AC4: `buildJsonLd` aceita `imageUrl` e `imageAlt` em `JsonLdOptions` e inclui `ImageObject` no schema `BlogPosting` quando `imageUrl` está presente
- [ ] AC5: Ghost gera `og:image` automaticamente a partir de `feature_image` (comportamento nativo do Ghost — validado via `<meta property="og:image">` no HTML do post publicado)
- [ ] AC6: Se a NuvemShop API falhar ou nenhum produto tiver imagem, o post é publicado normalmente sem `feature_image` — sem erro, sem bloqueio
