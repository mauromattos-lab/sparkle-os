# Story 5.2 — Gate de Aprovação no Cockpit

**Epic:** 5 — Content Engine: Automação e Integração
**Status:** Done
**Prioridade:** P1 — Blocker
**Executor:** @dev
**Estimativa:** M
**Depende de:** 5.1

---

## Descrição

Como Mauro, quero ver o post gerado pelo squad diretamente no Cockpit e aprovar com um clique, para que nada seja publicado sem minha revisão e o processo tome menos de 2 minutos do meu dia.

---

## Acceptance Criteria

- [ ] AC1: O Cockpit exibe um painel "Content Engine" com o post do dia quando disponível
- [ ] AC2: O painel mostra: título do post, meta description, primeiros 200 caracteres do corpo, bloco de conteúdo (cuidados/estilo/etc), imagem selecionada pela Vista (descrição), copy do pin Pinterest
- [ ] AC3: Dois botões presentes: **Aprovar** e **Rejeitar com nota**
- [ ] AC4: Ao clicar Aprovar → status muda para `aprovado`, pipeline de publicação é disparada (Stories 5.3 e 5.4)
- [ ] AC5: Ao clicar Rejeitar com nota → Mauro digita uma instrução curta, post volta para Lyra com o feedback
- [ ] AC6: Status do post é visível em tempo real no painel (gerando → aguardando → publicado)
- [ ] AC7: Histórico dos últimos 7 posts publicados visível no painel

---

## Escopo

**IN:**
- Novo painel no Cockpit existente (Epic 4)
- Preview do post (não o post completo — apenas o essencial para decidir)
- Botões de ação: Aprovar / Rejeitar com nota
- Status em tempo real
- Histórico simples (7 dias)

**OUT:**
- Editor de post no Cockpit (edição é feita via agente, não na UI)
- Analytics de performance
- Notificação push/email (Mauro verifica manualmente no início)

---

## Dependências

- Story 5.1 (Scheduler) — o post precisa estar sendo gerado
- Cockpit existente (Epic 4) — adicionar painel, não recriar

---

## Notas de UX

- Painel deve ser a primeira coisa que Mauro vê no Cockpit quando há algo pendente
- Indicador visual claro de "1 post aguardando aprovação"
- Preview deve ser suficiente para decidir — não precisa ler tudo

## Dev Notes (Aria — @architect)

### Integração no Cockpit existente (padrão Epic 4)

Adicionar estes arquivos seguindo o padrão dos painéis existentes:

```
packages/cockpit/src/
  routes/content-engine.ts           # handler do painel — renderiza HTML
  services/content-engine.service.ts # lê content_posts da DB via @sparkle-os/core
```

**Registrar em `packages/cockpit/src/index.ts`:**
```typescript
import { renderContentEngine } from './routes/content-engine.js';
app.post('/cockpit/content/approve/:id', handleApprove);
app.post('/cockpit/content/reject/:id', handleReject);
app.get('/cockpit/content', renderContentEngine);
```

**Fonte de dados:** tabela `content_posts` (criada na Story 5.1). O serviço lê via Drizzle da mesma forma que os outros serviços do Cockpit.

**Ações do painel:**
- `POST /cockpit/content/approve/:id` → atualiza status para `aprovado`, dispara pipeline de publicação (Stories 5.3 + 5.4)
- `POST /cockpit/content/reject/:id` → recebe nota, devolve para Lyra com feedback

**Histórico** (AC7): query `content_posts` WHERE `created_at >= NOW() - 7 days` ORDER BY `created_at DESC`

**Pattern de referência:** ver `packages/cockpit/src/routes/overview.ts` para estrutura exata de renderização HTML SSR.
