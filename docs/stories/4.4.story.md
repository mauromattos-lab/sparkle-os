# Story 4.4 — Painel do Núcleo Zenya

## User Story

Como Mauro, quero ver o estado operacional da Zenya sem precisar acessar o n8n ou o Chatwoot, para que eu saiba em segundos se a Zenya está funcionando normalmente ou se há algo que precisa da minha atenção.

## Acceptance Criteria

- [x] 1. Painel `/cockpit/zenya` exibe indicador de conectividade do n8n (verde = responde, vermelho = offline) — verificação via `GET http://localhost:5678/api/v1/health` ou `GET http://localhost:5678/`
- [x] 2. Número de workflows ativos no n8n exibido quando disponível — via `GET http://localhost:5678/api/v1/workflows?active=true&limit=1` (usando `totalCount` do header ou contagem da resposta)
- [x] 3. Status das stories da Zenya (Epic 2) listado: quantas Done, quantas InProgress
- [x] 4. SOP inline: se n8n offline, texto explica a Mauro o que verificar ("Verifique se o serviço n8n está rodando na VPS")
- [x] 5. Painel acessível via navegação lateral do shell

## Complexidade
**M (Médio)** — Dois sources de dados (n8n HTTP + story parser), ambos simples individualmente.

## Riscos

- **n8n API Key:** `GET /api/v1/workflows` requer autenticação (`X-N8N-API-KEY`). Para o health check de conectividade, tentar primeiro `GET /` ou `GET /healthz` que são públicos. Documentar no `.env.example` a variável `N8N_API_KEY` mesmo que usada só quando disponível.
- **n8n não rodando localmente:** Em desenvolvimento local (máquina de Mauro), o n8n está na VPS, não em localhost. A URL correta pode ser via SSH tunnel ou a URL da VPS. Usar variável de ambiente `N8N_URL` em vez de hardcode.
- **Timeout:** Se n8n não responde, a request não deve travar o cockpit. Usar timeout de 5 segundos no fetch.


Estratégia de health check em camadas:
```typescript
// Nível 1: ping básico (sem auth) — verifica se n8n responde
const res = await fetch(`${N8N_URL}/healthz`, { signal: AbortSignal.timeout(5000) });
// Se 200 → verde

// Nível 2: contagem de workflows (requer API key — opcional)
if (N8N_API_KEY) {
  const wf = await fetch(`${N8N_URL}/api/v1/workflows?active=true`, {
    headers: { 'X-N8N-API-KEY': N8N_API_KEY },
    signal: AbortSignal.timeout(5000)
  });
  // wf.data.length → número de workflows ativos
}
```
