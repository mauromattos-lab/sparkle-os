## Descrição

**As a** gestor de atendimento,
**I want** que quando a Zenya escalar uma conversa para humano, a etiqueta correta apareça automaticamente no WhatsApp Business do atendente,

## Dev Notes

### Estrutura do novo módulo `zapi-labels.ts`

```typescript
export interface ZApiCredentials {
  instanceId: string;
  token: string;
  clientToken: string;
  labels?: {
    humano?: string; // ID numérico retornado pela Z-API ao criar a etiqueta
  };
}

export async function zapiAddLabel(
  phone: string,
  labelId: string,
  creds: ZApiCredentials,
): Promise<void> {
  const url = `https://api.z-api.io/instances/${creds.instanceId}/token/${creds.token}/tags-add`;
  const res = await fetch(url, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json', 'client-token': creds.clientToken },
    body: JSON.stringify({ phone: normalizePhone(phone), tag: labelId }),
  });
  if (!res.ok) throw new Error(`Z-API tags-add failed (${res.status})`);
}

export async function zapiRemoveLabel(
  phone: string,
  labelId: string,
  creds: ZApiCredentials,
): Promise<void> {
  const url = `https://api.z-api.io/instances/${creds.instanceId}/token/${creds.token}/tags-remove`;
  const res = await fetch(url, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json', 'client-token': creds.clientToken },
    body: JSON.stringify({ phone: normalizePhone(phone), tag: labelId }),
  });
  if (!res.ok) throw new Error(`Z-API tags-remove failed (${res.status})`);
}
```

### Normalização do phone

O phone vem do Chatwoot no formato `+5531999998888`. A Z-API espera sem o `+`.
Reutilizar a lógica já existente no `dist/integrations/zapi.js` ou implementar inline:
```typescript
function normalizePhone(phone: string): string {
  return phone.replace(/^\+/, '');
}
```

### Lógica de `escalarHumano` com degradação graciosamente

```typescript
escalarHumano: tool({
  execute: async ({ resumo_conversa, motivo }) => {
    const params = chatwootParams();
    await addLabel(params, 'agente-off'); // Chatwoot — crítico

    // Z-API labels — não-crítico, degrada graciosamente
    try {
      const zapCreds = await getCredentialJson<ZApiCredentials>(tenantId, 'zapi');
      const labelId = zapCreds.labels?.humano;
      if (labelId) {
        await zapiAddLabel(ctx.phone, labelId, zapCreds);
      }
    } catch (err) {
      console.warn(`[zenya] zapiAddLabel falhou (non-critical) — ${err}`);
    }

    // ... log e return existentes
  },
}),
```

### Cleanup — como obter o phone da conversa

A resposta do endpoint `GET /conversations?labels[]=agente-off` do Chatwoot inclui o campo `meta.sender.phone_number`. O tipo em `getAgenteOffConversations()` precisa ser expandido:

```typescript
// agente-off-cleanup.ts — tipo expandido
interface AgenteOffConversation {
  id: number;
  labels: string[];
  last_activity_at: number;
  meta?: {
    sender?: {
      phone_number?: string | null;
    };
  };
}

// Extração do phone dentro do loop de cleanup:
const phone = conv.meta?.sender?.phone_number ?? null;
if (phone && labelId) {
  await zapiRemoveLabel(phone, labelId, zapCreds).catch((err) => {
    console.warn(`[agente-off-cleanup] zapiRemoveLabel falhou (non-critical) — ${err}`);
  });
}
```

O campo `meta.sender.phone_number` já é retornado pelo Chatwoot na listagem de conversas — sem chamadas HTTP adicionais.

### Script `setup-zapi-labels.mjs`

Uso:
```bash
node scripts/setup-zapi-labels.mjs <tenantId> humano
```

Fluxo:
1. Carrega credenciais Z-API do Supabase para o tenant
2. Faz `GET /tags` — verifica se etiqueta com esse nome já existe
3. Se não existe → `POST /create-tag` com o nome
4. Obtém o `id` retornado
5. Atualiza o JSON de credenciais no Supabase adicionando `labels.humano = id`
6. Re-encripta e salva

## Riscos

- **Instância não-Business / não-Multi-Devices:** Z-API retorna erro — mitigado pela degradação graciosamente (warn, não throw)
- **Credenciais sem `labels` configurado:** feature simplesmente não ativa — zero impacto em tenants existentes
- **Race condition no cleanup:** se o cleanup remover a etiqueta Z-API mas falhar no Chatwoot (ou vice-versa) — aceitável, ambas as operações são idempotentes
