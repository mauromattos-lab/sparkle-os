# Story 9.2 — Publicação Automática: Feature Image + Ghost

**Epic:** [Epic 9](./EPIC-9-INDEX.md)
**Status:** Draft
**Prioridade:** P1
**Depende de:** 9.1
**Executor:** @dev
**Estimativa:** M (3-4h)

## Descrição

Adicionar ao workflow n8n os steps de publicação automática: resolução de feature image via NuvemShop API + processamento sharp (1:1 → 1200×630 hero) + publicação no Ghost Admin API + atualização de `posts-history.md`.

Reutiliza a lógica já implementada em `process-feature-images.mjs` e `patch-varied-images.mjs` (Story 8.2), encapsulada em um script Node.js invocável pelo n8n via `Execute Command node`.

---

## Acceptance Criteria

**AC1 — Resolução de feature image (Step 7):**
- Recebe `imageDesc` do output da Vista (ou deriva do briefing/bloco_conteudo se Vista não for usada nesta story)
- Chama NuvemShop API buscando produto mais relevante para o tópico
- Seleciona imagem com modelo confirmado (índice de imagem verificado, não apenas img_* prefix)
- Fallback: se não encontrar produto relevante, usa produto com melhor lifestyle photo disponível

**AC2 — Processamento sharp (Step 8):**
- Invoca script Node.js no VPS: `node /opt/sparkle/packages/content-engine/scripts/auto-publish-post.mjs`
- Script recebe via stdin ou args: post em Markdown, imageUrl, imageAlt, slug, tags
- Faz download da imagem original (1:1) do NuvemShop
- Processa para 1200×630: fundo desfocado (blur 30, brightness 0.6) + imagem centralizada
- Salva processado em /tmp para upload imediato

**AC3 — Upload da imagem ao Ghost (passo interno do script):**
- POST `/ghost/api/admin/images/upload/` com FormData (multipart/form-data)
- Recebe URL Ghost-hosted da imagem processada
- JWT renovado a cada operação (expiração 5min)

**AC4 — Publicação do post no Ghost (Step 9):**
- POST `/ghost/api/admin/posts/` com status `published`
- Campos obrigatórios: `title`, `slug`, `html` (convertido de Markdown), `feature_image`, `feature_image_alt`
- Tags aplicadas automaticamente baseadas no `bloco_conteudo` do briefing:
  - cuidados → `cuidados`, `semi-joias`
  - qualidade → `qualidade`, `semi-joias`
  - estilo → `estilo`, `semi-joias`
  - materiais → `materiais`, `semi-joias`
  - ocasioes → `ocasioes`, `semi-joias`
- `published_at`: timestamp atual

**AC5 — Atualização de posts-history.md (Step 10):**
- Após publicação bem-sucedida, script faz append em `squads/aeo-squad-plaka/data/posts-history.md`:
  ```
  | YYYY-MM-DD | Título do Post | bloco | /slug/ | URL |
  ```
- Git commit automático: `feat(aeo): post diário YYYY-MM-DD — {título}`
- Git push para origin/main via SSH (VPS já tem chave configurada)

**AC6 — Idempotência:**
- Se post com mesmo slug já existe no Ghost: não cria duplicado, faz PATCH
- Se `posts-history.md` já tem a data: não duplica a linha

## Notas Técnicas

**Script `auto-publish-post.mjs` — interface:**
```bash
node auto-publish-post.mjs \
  --title "Título do Post" \
  --slug "slug-do-post" \
  --content-file /tmp/post.md \
  --topic "choker" \
  --bloco "cuidados"
```

**Dependências do script:**
- `sharp` — já instalado em `packages/content-engine/scripts/node_modules/`
- `marked` — para conversão Markdown → HTML
- `node-fetch` — Node 18+ tem fetch nativo

**Markdown → HTML no Ghost:**
Ghost aceita `html` (preferível) ou `lexical`. Usar `marked` para converter:
```javascript
import { marked } from 'marked';
const html = marked(markdownContent);
```

**Tags no Ghost:**
Tags já criadas no Epic 8 (Story 8.1). Referenciar por slug:
```json
"tags": [{ "slug": "cuidados" }, { "slug": "semi-joias" }]
```

**Git automático — script shell wrapper:**
```bash
cd /opt/sparkle
git add squads/aeo-squad-plaka/data/posts-history.md
git commit -m "feat(aeo): post diário $(date +%Y-%m-%d) — $TITLE"
git push origin main
```
