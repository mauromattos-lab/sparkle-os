# Story 9.2 — Script daily-pipeline.mjs: Imagem + Publicação Ghost

**Epic:** [Epic 9](./EPIC-9-INDEX.md)
**Status:** Draft
**Prioridade:** P1
**Depende de:** 9.1
**Executor:** @dev
**Estimativa:** M (3-4h)

---

## Descrição

Completar o `daily-pipeline.mjs` com os steps 7 a 10: resolução de feature image via NuvemShop (com índice de imagem com modelo confirmado), processamento sharp 1:1→1200×630, publicação no Ghost Admin API, e commit automático do `posts-history.md` via `git-auto-commit` no workflow.

Reutiliza a lógica já validada em `patch-varied-images.mjs` e `process-feature-images.mjs` (Epic 8), consolidada no script único do pipeline.

---

## Acceptance Criteria

**AC1 — Resolução de feature image (Step 7):**
- Deriva query de busca a partir do `bloco_conteudo` do briefing (Sage):
  ```
  cuidados   → "choker" (nth=0, imgIdx=1)
  qualidade  → "estrelinhas" (nth=0, imgIdx=1)
  estilo     → "colar choker disco" (nth=0, imgIdx=1)
  materiais  → "corrente bicolor" (nth=0, imgIdx=1)
  tendencias → "galax" (nth=0, imgIdx=0)
  ocasioes   → "bolinhas ouro" (nth=0, imgIdx=2)
  ```
- Fallback se bloco não mapeado: busca pelo tópico do briefing com Sharp analysis para confirmar modelo
- Retorna `{ imageUrl, productName }`

**AC2 — Processamento sharp (Step 8):**
- Baixa imagem original do NuvemShop (1:1)
- Processa para 1200×630:
  - Fundo: resize cover + blur(30) + brightness(0.6)
  - Foreground: resize height 590 (withoutEnlargement) + composite centrado
- Output: Buffer JPEG quality 88
- Sharp importado via `createRequire` (padrão já estabelecido nos scripts do Epic 8)

**AC3 — Upload da imagem ao Ghost (Step 9a):**
- `POST /ghost/api/admin/images/upload/` com FormData multipart
- JWT renovado a cada operação (buildJwt do `_publish-helpers.mjs`)
- Retorna URL Ghost-hosted da imagem

**AC4 — Publicação do post no Ghost (Step 9b):**
- Gera slug a partir do título: função `slugify` do `_publish-helpers.mjs`
- Converte Markdown → HTML usando `marked` (instalar como dependência)
- `POST /ghost/api/admin/posts/` com:
  ```json
  {
    "status": "published",
    "title": "...",
    "slug": "...",
    "html": "...",
    "feature_image": "<ghost-url>",
    "feature_image_alt": "<productName> — Plaka Acessórios",
    "tags": [{ "slug": "<bloco>" }, { "slug": "semi-joias" }],
    "published_at": "<ISO timestamp>"
  }
  ```
- Tags já existem no Ghost (criadas no Epic 8)
- Se post com mesmo slug já existir: `PUT` (update) em vez de `POST` (idempotente)

**AC5 — Atualização de posts-history.md (Step 10):**
- Script faz append da nova linha no arquivo:
  ```
  | YYYY-MM-DD | Título do Post | bloco | slug | https://blog.plakaacessorios.com/slug/ |
  ```
- Arquivo salvo localmente no runner (o `git-auto-commit` do workflow fará o commit)

**AC6 — git-auto-commit no workflow:**
- Adicionar ao `.github/workflows/plaka-daily-content.yml` após o step de execução:
  ```yaml
  - uses: stefanzweifel/git-auto-commit-action@v5
    with:
      commit_message: 'feat(aeo): post diário ${{ env.POST_DATE }} — ${{ env.POST_TITLE }}'
      file_pattern: 'squads/aeo-squad-plaka/data/posts-history.md'
  ```
- `POST_DATE` e `POST_TITLE` exportados pelo script via `$GITHUB_ENV`

**AC7 — Idempotência:**
- Se post com mesmo slug já existe no Ghost: PATCH em vez de POST
- Se linha com mesma data já existe em posts-history.md: não duplica
