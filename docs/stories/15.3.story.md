## Dev Notes

### Por que nova §9.0 em vez de reorganizar todo o §9?

§9.1-§9.8 atuais (pré-reqs Chatwoot, prompt canônico, seed, dry-run, credenciais, smoke, cutover) são PASSOS. §9.0 é META-PASSO: "qual desses passos aplicam pra mim?". Adicionar como §9.0 mantém numeração existente estável (zero quebra de links internos) e só inclui nova entrada no topo.

### Tabela taxonomia

Formato em AC1 é **proposta**. @dev pode ajustar colunas se alguma não fizer sentido (ex: coluna "smoke template" pode virar nota se a variante é óbvia pelo tipo). O essencial é a informação: pra cada tipo, quais passos e quais variantes.

### Link forward vs link direto

`TENANT-REFINEMENT-PLAYBOOK.md` fica no mesmo diretório `docs/zenya/` → link relativo `./TENANT-REFINEMENT-PLAYBOOK.md` funciona.

`smoke-template.mjs` fica em `packages/zenya/scripts/` → a partir de `docs/zenya/TENANT-PLAYBOOK.md`, o link relativo é `../../packages/zenya/scripts/smoke-template.mjs`. Conferir por testar no preview GitHub.

### Nota sobre ordem de execução

Se a equipe preferir paralelizar 15.2 e 15.3, o @dev de 15.3 pode começar com TBDs e abrir fix-story pequena após. Mas sequencial 15.1 → {15.2 || 15.3} é mais limpo e adiciona pouco atrito (Story 15.3 é pequena).
