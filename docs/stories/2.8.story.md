# Story 2.8 — Base de Conhecimento Operacional da Zenya

## User Story
Como agente AIOS, quero uma base de conhecimento estruturada sobre a operação da Zenya — como o sistema funciona no dia a dia, quais são as situações comuns, como cada fluxo se comporta em prática — para que qualquer agente futuro entre em contexto rapidamente e o Collective Brain tenha insumos de qualidade desde o início (FR15).

### Situações a Documentar (base da Story 2.1)
Derivadas do mapa de fluxos — cada situação mapeia para um ou mais fluxos:
- Cliente envia texto → `01.` → AI Agent → `07.`
- Cliente envia áudio → `01.` (transcribe via Whisper) → AI Agent → `07.`
- Cliente envia arquivo → `01.` → `02.` (Drive) → `07.`
- Cliente solicita agendamento → `01.` → AI Agent tool → `03.` → `04.`
- Agendamento precisa ser atualizado → `04.1`
- Cliente solicita boleto/cobrança → `01.` → AI Agent tool → `06.`
- Agente não consegue resolver → `01.` → AI Agent tool → `05.` (escalação)
- Lembrete de agendamento → `11.` (schedule)
- Cliente liga → `12.` (webhook Chatwoot)
- Cliente não responde → `13.` (inativo — reengajamento)
