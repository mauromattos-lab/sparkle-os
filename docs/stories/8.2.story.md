# Story 8.2 — Feature Images: Backfill e Product Enricher no Fluxo Manual

## Descrição

Ativar a resolução de feature images via `fetchRelevantProductImageUrl` (product-enricher) no fluxo de publicação manual do squad, e aplicar backfill nos 8 posts existentes que foram publicados sem imagem. Sem feature image: Open Graph quebrado, schema ImageObject inválido, Pinterest inoperante.

---

## Acceptance Criteria

**AC1 — Função `fetchRelevantProductImageUrl` integrada nos scripts manuais:**
- [x] Criar helper `packages/content-engine/scripts/_publish-helpers.mjs` com função `resolveFeatureImage(title, topic)` que chama `fetchRelevantProductImageUrl` via import do dist
- [x] Helper exporta também `buildJwt` e `slugify` para eliminar duplicação nos scripts existentes
- [x] Documentar: se NuvemShop indisponível, retorna `null` e publicação segue sem imagem (graceful degradation — padrão já existente no `publication-orchestrator.ts`)

**AC2 — Script de backfill para os 8 posts existentes:**
- [x] Script `packages/content-engine/scripts/backfill-feature-images.mjs` criado
- [x] Para cada post (por slug): GET para obter ID do post, chama `resolveFeatureImage`, PATCH com `feature_image` e `feature_image_alt`
- [x] Log: para cada post, exibe produto selecionado e URL da imagem ou `[sem imagem — NuvemShop indisponível]`
- [x] Idempotente: skip posts que já têm `feature_image` definida

**AC3 — Schema JSON-LD atualizado no backfill:**
- [x] O PATCH via Admin API inclui `codeinjection_head` atualizado com `ImageObject` válido dentro do `BlogPosting`
- [x] Merge inteligente: preserva FAQPage e outros schemas existentes; localiza BlogPosting e injeta campo `image`

**AC4 — Novos scripts de publicação incluem feature image:**
- [x] Helper `_publish-helpers.mjs` serve como template de referência — importar `resolveFeatureImage` antes do POST
- [ ] Documentar no README do squad: ao criar script manual, sempre chamar `resolveFeatureImage` antes do POST

**AC5 — Open Graph verificável:**
- [ ] Após backfill, pelo menos 1 post verificado via `curl -s https://blog.plakaacessorios.com/{slug}/ | grep og:image` retorna URL de imagem válida

---

## Scope

**IN:**
- Helper de publicação reutilizável
- Backfill de feature images nos 8 posts via PATCH
- Atualização do JSON-LD com ImageObject
- Documentação de uso

**OUT:**
- Pinterest publisher automático (depende de API token não configurado)
- Mudança no `pipeline-runner.ts` (fluxo automatizado já funciona)
- Geração de imagens via AI/DALL-E

## Risks

- **Risco:** NuvemShop API indisponível durante backfill → Mitigação: graceful degradation, script loga warning e skip sem falhar
- **Risco:** PATCH de `codeinjection_head` sobrescreve o JSON-LD original → Mitigação: GET post antes, reconstruir JSON-LD completo com todos os dados originais + imageUrl novo
- **Rollback:** PATCH com `feature_image: null` e `codeinjection_head` original restaura estado anterior
