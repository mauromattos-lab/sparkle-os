## Acceptance Criteria

**AC1 — Notificação de sucesso (após Step 9):**
- Chamada `POST` para Z-API após publicação bem-sucedida no Ghost
- Mensagem:
  ```
  ✅ *Post publicado — Plaka Blog*

  📝 {título}
  🏷 {bloco_conteudo} · {data}
  🔗 https://blog.plakaacessorios.com/{slug}/

  Gerado automaticamente pelo squad AEO.
  ```

**AC2 — Notificação de escalação (Step 6):**
- Chamada Z-API quando `veredicto == "ESCALADO"`
- Mensagem:
  ```
  ⚠️ *Post precisa de revisão — Plaka Blog*

  Tópico: {topico do briefing}
  Feedback Rex: {feedback da última iteração}

  Revisar em: github.com/{repo}/actions
  ```

**AC3 — Notificação de falha técnica (via GitHub Actions nativo):**
- Erros técnicos (Claude API down, Ghost indisponível) fazem `process.exit(1)`
- GitHub Actions envia email automático para o owner do repo ao falhar
- Nenhuma lógica adicional necessária para esse caso

**AC4 — Credenciais via GitHub Secrets:**
- `ZAPI_INSTANCE_ID`, `ZAPI_TOKEN`, `MAURO_PHONE` já definidos como secrets
- Função `sendWhatsApp(message)` encapsula a chamada Z-API

## Notas Técnicas

**Função sendWhatsApp:**
```javascript
async function sendWhatsApp(message) {
  const instanceId = process.env.ZAPI_INSTANCE_ID;
  const token = process.env.ZAPI_TOKEN;
  const phone = process.env.MAURO_PHONE;
  if (!instanceId || !token || !phone) {
    console.warn('[notify] Z-API não configurado — pulando notificação');
    return;
  }
  await fetch(
    `https://api.z-api.io/instances/${instanceId}/token/${token}/send-text`,
    {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ phone: `55${phone}`, message }),
    }
  );
}
```

Função gracefully degradada: se secrets não configurados, loga warning e continua sem falhar o pipeline.
