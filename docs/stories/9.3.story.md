# Story 9.3 — Notificações: WhatsApp para Sucesso e Escalação

**Epic:** [Epic 9](./EPIC-9-INDEX.md)
**Status:** Draft
**Prioridade:** P2
**Depende de:** 9.1
**Executor:** @dev
**Estimativa:** S (1-2h)

---

## Descrição

Adicionar notificações automáticas para o Mauro via WhatsApp usando Z-API (já integrado no Zenya). Dois cenários: (1) post publicado com sucesso — resumo com título e URL; (2) escalação necessária — Rex rejeitou 2x e precisa de revisão humana.

---

## Acceptance Criteria

**AC1 — Notificação de sucesso (Step 11):**
- Após publicação bem-sucedida no Ghost (Step 9 da Story 9.2)
- Envia mensagem WhatsApp para o número do Mauro via Z-API
- Formato da mensagem:
  ```
  ✅ *Post publicado — Plaka Blog*

  📝 {título do post}
  🏷 {bloco_conteudo} · {data}
  🔗 {URL completa do post}

  Gerado automaticamente pelo squad AEO.
  ```

**AC2 — Notificação de escalação (Step 6):**
- Quando Rex rejeita 2x (veredicto = ESCALADO)
- Envia alerta WhatsApp para o Mauro
- Formato da mensagem:
  ```
  ⚠️ *Post precisa de revisão — Plaka Blog*

  Tópico: {tópico do briefing}
  Problema: {feedback do Rex na 2ª iteração}

  Acesse o n8n para revisar: http://187.77.37.88:5678
  ```

**AC3 — Notificação de falha técnica:**
- Se o workflow falha por erro técnico (API timeout, script crash)
- Envia alerta simplificado:
  ```
  🔴 *Pipeline AEO falhou — {data}*

  Erro: {mensagem do erro}
  Verificar logs: http://187.77.37.88:5678
  ```

**AC4 — Reutilizar integração Z-API existente:**
- Usar as mesmas credenciais Z-API já configuradas no Zenya
- Endpoint: Z-API `/send-text` com instanceId e token do `.env`
- Número de destino: número do Mauro (configurado como variável n8n `MAURO_PHONE`)

---

## Escopo

**IN:**
- 3 tipos de notificação (sucesso, escalação, falha técnica)
- Integração com Z-API via HTTP node do n8n
- Variáveis: `ZAPI_INSTANCE_ID`, `ZAPI_TOKEN`, `MAURO_PHONE`

**OUT:**
- Notificações para outros canais (Slack, email)
- Notificações para o time além do Mauro
- Dashboard de métricas de publicação

---

## Notas Técnicas

**Z-API endpoint:**
```
POST https://api.z-api.io/instances/{INSTANCE_ID}/token/{TOKEN}/send-text
Content-Type: application/json

{
  "phone": "55{MAURO_PHONE}",
  "message": "..."
}
```

**Variáveis no n8n:**
As credenciais Z-API já existem no ambiente Zenya. Reutilizar via n8n Credentials Store ou variáveis de ambiente do VPS.
