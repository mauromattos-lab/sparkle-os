# Story 6.4 — Deprecação do NuvemShop Blog Publisher

**Epic:** 6 — Canal de Blog AEO (Ghost CMS)
**Status:** Done
**Prioridade:** P3
**Executor:** @dev
**Estimativa:** S
**Depende de:** 6.2 ✅ Done

---

## Descrição

Como desenvolvedor do SparkleOS, quero remover o código morto do `nuvemshop-publisher.ts` que tentava publicar em um endpoint de blog inexistente na NuvemShop, para que o codebase fique limpo e não cause confusão sobre qual publisher está ativo. O canal de blog agora é o Ghost CMS (Story 6.2).

---

## Acceptance Criteria

- [x] AC1: A função `publishToNuvemShop` é removida de `nuvemshop-publisher.ts`
- [x] AC2: As funções auxiliares usadas exclusivamente por `publishToNuvemShop` (`getNuvemShopImageUrl`) são removidas
- [x] AC3: A função `markdownToHtml` é **mantida** em `nuvemshop-publisher.ts` — ela é importada pelo `ghost-publisher.ts`
- [x] AC4: O import de `publishToNuvemShop` no `publication-orchestrator.ts` (se existir) é removido
- [x] AC5: Os testes de `publishToNuvemShop` e `getNuvemShopImageUrl` são removidos do test suite — apenas `markdownToHtml` permanece testado
- [x] AC6: `npm run typecheck` passa após as remoções
- [x] AC7: O arquivo `nuvemshop-publisher.ts` permanece no codebase (não é deletado) — contém `markdownToHtml` ainda em uso

---

## Escopo
