## Descrição

Como Mauro, quero que o post publicado no Ghost use uma imagem de destaque (`feature_image`) do produto **semanticamente mais relevante** para o tópico do post — e não simplesmente o primeiro produto do catálogo — para que o `og:image`, o `alt text` e o `ImageObject` do schema JSON-LD reforcem coerentemente o conteúdo do artigo, aumentando a precisão dos sinais visuais para motores de IA (Google AIO, Perplexity, ChatGPT Search).

**Problema atual (Story 6.7):** `fetchFirstProductImageUrl()` retorna sempre o primeiro produto publicado da lista NuvemShop, independente do tópico do post. Resultado: posts sobre colares recebem imagem de anel — incoerência semântica que penaliza o sinal `ImageObject` no schema AEO.

**Solução:** Usar o modelo OpenAI já configurado (`gpt-4o-mini`) para selecionar, a partir da lista de produtos, aquele mais relevante para o tópico/título do post. Fallback automático para o primeiro produto se o LLM falhar (graceful degradation — sem bloqueio à publicação).

**Alt text adicional:** O `feature_image_alt` deve descrever o **produto selecionado** (ex: `"Colar Gotas Eye semi joia banhado à prata — Plaka Acessórios"`), não a descrição genérica do Vista — garantindo coerência entre imagem, alt text e conteúdo.

### Prompt LLM (conciso e determinístico)

```typescript
function buildSelectionPrompt(
  postTitle: string,
  postTopic: string | undefined,
  productNames: string[],
): string {
  const topicLine = postTopic ? `\nTópico: ${postTopic}` : '';
  const list = productNames.map((n, i) => `${i}: ${n}`).join('\n');
  return (
    `Título do post: ${postTitle}${topicLine}\n\n` +
    `Produtos disponíveis:\n${list}\n\n` +
    `Responda APENAS com o número (índice) do produto mais relevante para o post. ` +
    `Responda somente com um número inteiro.`
  );
}
```
